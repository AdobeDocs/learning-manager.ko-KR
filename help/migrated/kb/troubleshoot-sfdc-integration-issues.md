## SFDC 내보내기 실패 문제 해결(2~3시간 이상 내보내기 없음)

SFDC 내보내기가 **2~3시간**&#x200B;을 초과하여 성공적으로 완료되지 않은 경우 아래 단계를 사용하여 문제를 파악하고 파악하십시오.

1. **Salesforce 홈** 페이지로 이동합니다.
2. **빠른 찾기** 검색 상자에 **`Bulk Data Load Jobs`**&#x200B;을(를) 입력하고 엽니다.
3. 페이지에 **최근 7일**&#x200B;에 대한 대량 데이터 로드 작업이 표시됩니다.
4. **Adobe Learning Manager(ALM) 개체**&#x200B;와 관련된 작업을 찾습니다.
5. 조사할 특정 작업을 클릭합니다.
6. 작업 세부 정보 페이지의 **아래쪽**(으)로 스크롤하여 **오류 메시지** 및 추가 세부 정보를 봅니다.

대부분의 경우, 근본 원인은 **SFDC에서 필드 수준 권한이 부족함**&#x200B;입니다. 실패한 필드 이름은 다음과 같을 수 있습니다.

- `cp_Module_ID`
- 또는 ALM 사용자 지정 개체와 관련된 다른 `cp_...` 필드입니다.

이러한 필드가 오류에 나타나면 다음 섹션으로 진행합니다.

## SFDC의 ALM 사용자 정의 개체 필드에 대한 필드 권한 부여

**필드 접근성** 기능을 사용하여 특정 필드(예: `cp_Module_ID`)에 대한 권한 문제를 수정하십시오.

### 단계

1. **Salesforce 홈** 페이지로 이동합니다.
2. **빠른 찾기** 검색 상자에 **`Field Accessibility`**&#x200B;을(를) 입력하고 엽니다.
3. 개체 목록에서 관련 **보고서 유형/개체**&#x200B;를 선택합니다.
   - 예: 학습자 성적 증명서(LT) 보고서의 경우 `cp_LearnerTranscript_xxxx_xxxx`.
4. 다음 페이지에서 **&quot;필드 기준 보기&quot;**&#x200B;를 선택합니다.
5. **필드 드롭다운**&#x200B;에서 내보내기 오류에 나타난 필드를 선택합니다
   - 예: `cp_Module_ID`.
6. 액세스 매트릭스에서 **시스템 관리자** 프로필(필요한 경우 기타 관련 프로필)에 대한 **&quot;숨김&quot;** 항목을 클릭합니다.
7. 다음 페이지에서 다음을 수행합니다.
   - 필요한 경우 **&quot;표시&quot;**&#x200B;을(를) 활성화합니다.
   - 페이지 하단의 **저장**&#x200B;을 클릭합니다.
8. 저장 후 필드 접근성 보기로 돌아갑니다. 이제 **시스템 관리자**(및 업데이트한 다른 프로필)에 대해 이 필드가 **&quot;편집 가능&quot;**&quot;으로 표시됩니다.

## 영향을 받는 모든 필드에 대해 반복하고 내보내기를 다시 시도하십시오

1. **대량 데이터 로드 작업**&#x200B;에서 권한 문제가 있는 것으로 보고된 **각 필드**&#x200B;에 대한 섹션 2의 필드 접근성 단계를 **반복**&#x200B;합니다.
2. 문제가 있는 모든 필드에 적절한 가시성 및 편집 권한이 부여되면 다음을 수행하십시오.
   - **Adobe Learning Manager**(으)로 돌아갑니다.
   - **SFDC 커넥터** 구성에서 **내보내기를 다시 시도하십시오**.


## 중요 참고 사항 및 제한 사항

통합을 설계하거나 문제를 해결할 때 SFDC-ALM 관련 세부 사항을 염두에 두십시오.

### SFDC에 자동 개체/필드 생성 기능 없음

- **SFDC 커넥터가 Salesforce**&#x200B;에서 새 개체 또는 필드를 만들지 않습니다.
- **새 필드가 ALM**&#x200B;에 추가되고 SFDC에 표시되게 하려는 경우:
   - SFDC에서 수동으로 **해당 사용자 지정 필드를 만듭니다**.
   - 커넥터 구성의 **적절한 ALM 필드**&#x200B;에 SFDC 사용자 지정 필드를 **매핑**&#x200B;합니다.
   - 새 필드에 **적절한 필드 수준 권한**&#x200B;이 있는지 확인하십시오(섹션 2 사용).

### 사용자 정의 도메인이 있는 ALM 계정의 콜백 URL

- ALM 계정이 **사용자 지정 도메인**&#x200B;을 사용하는 경우, 엔지니어링 팀은 SFDC 커넥터의 OAuth 프로덕션 앱에 대한 **콜백 URL 허용 목록**&#x200B;에 **해당 사용자 지정 도메인을 추가**&#x200B;해야 합니다.
- 이 작업은 엔지니어링에 대한 **Jira 요청**&#x200B;을 통해 수행됩니다.
- 이를 수행하지 않으면 커넥터에 대한 OAuth 흐름이 실패할 수 있습니다.

### 시간대 제한(UTC만 해당)

- SFDC 조직이 UTC **가 아닌**&#x200B;시간대를 사용하는 경우, 동기화 중에 **완료 및 등록 데이터가 누락될 수 있습니다**.
- 이유: ALM **은(는) 현재 SFDC 통합에 대한 UTC 시간대**&#x200B;만 지원합니다.
- 내부 참조: 추가 시간대를 지원하기 위해 PAPI-24525이 발생했습니다.

### 필터 데이터 유형 제한(선택 목록 및 체크리스트)

- ALM은 **선택 목록 필드로 만든 SFDC 필터** 가져오기를 지원합니다.
- ALM **은(는) 체크리스트/다중 선택 확인란 필드를 기반으로 하는** 필터를 지원하지 않습니다.
- 이유: ALM **SFDC 필터에 대한 문자열 데이터 형식**&#x200B;만 지원합니다.
