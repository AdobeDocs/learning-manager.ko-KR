# Learning Manager 설정을 위한 모범 사례 가이드

이 문서에서는 조직에서 Adobe Learning Manager를 설정하고 구성할 때 권장하는 방법을 소개합니다. Learning Manager 팀에서는 응용 프로그램 시작을 위해 단계별 접근 방식을 제시합니다. 특정 순서로 모든 단계를 따라야 하는 것은 아닙니다.

이러한 단계는 조직 설정에 따라 한 명 이상의 개인이 수행하는 세 가지 역할로 수행할 수 있습니다. 세 가지 역할은 다음과 같습니다.

1. **IT 관리자** - IT 관리자는 조직의 Learning Manager 응용 프로그램과 관련된 활성화 또는 통합 관련 활동을 수행합니다. 또한, IT 관리자는 단일/다수 사용자를 추가하고 통합 책임자의 역할을 수행할 수도 있습니다.
1. **작성자** - 작성자는 조직의 학습 요구 사항에 필요한 학습 콘텐츠를 만듭니다. 조직의 교육 또는 학습 콘텐츠 작성자는 Learning Manager 응용 프로그램에 필요한 기본 콘텐츠를 만들기 시작할 수 있습니다.
1. **Learning Manager 관리자** - Learning Manager 응용 프로그램 관리자는 구성 및 설정 작업을 수행합니다. 일부 회사에서는 IT 책임자가 Learning Manager 응용 프로그램 관리자 역할을 수행할 수도 있습니다.

아래에 제공된 인포그래픽을 통해 단계 및 해당 작업에 대한 개요를 볼 수 있습니다.

![](assets/learning-manager-getting-started.jpg)

이 단계에서는 조직에서 라이센스 키를 이미 받았고 Learning Manager 계정을 활성화한 상태라고 가정합니다. 인포그래픽에서 언급한 바와 같이, 세 가지 과정은 다음과 같습니다.

## 1단계 - 기술 설정(IT 관리자) {#phase1technicalsetupitadministrator}

트랙 1에서 조직의 IT 책임자는 Learning Manager 응용 프로그램에서 통합 책임자 역할로 전환하고 다음과 같은 활성화 및 통합을 수행할 수 있습니다.

### 활성 필드를 활성화/추가(Learning Manager 책임자) {#enableaddactivefieldscaptivateprimeadministrator}

관리자는 등록하는 동안 사용자가 제공하는 활성 필드 외에도 더 많은 활성 필드를 추가할 수 있습니다. 또한, 관리자는 사용자 필드를 사용하거나 활성화/비활성화할 수도 있습니다. 이러한 활성 필드 값은 사용자 계정과 연결된 다양한 데이터 소스의 메타데이터에서 생성됩니다. 자세한 내용은 [활성 필드 도움말](feature-summary/add-users-user-groups.md#active-fields)을 참조하십시오.

### 단일 인증(SSO) {#singlesignonsso}

Adobe ID 또는 단일 인증을 사용하여 Learning Manager 응용 프로그램에 접속할 수 있습니다. 단일 인증은 사용자가 한 번만 인증하면 여러 응용 프로그램에 다시 접속할 수 있도록 하는 구조입니다. 조직에 이 구성이 꼭 필요하지는 않습니다. 조직에 SAML 2.0 기반 SSO 제공자가 있으면 이를 이용하여 Learning Manager 응용 프로그램을 구성할 수 있습니다. 구성에는 조직 레벨과 Learning Manager 응용 프로그램이 필요합니다. SSO를 사용하도록 선택했다면 Adobe 지원팀에 문의하여 구성 지침을 받으십시오. 자세한 내용은 [설정 도움말](feature-summary/settings.md)을 참조하세요.

### 일괄 사용자 가져오기 {#bulkimportofusers}

조직에 사용자가 다수인 경우 .CSV 파일을 이용하여 모든 사용자를 Learning Manager 응용 프로그램으로 한 번에 가져올 수 있습니다. 이 작업을 수행하기 전에 조직의 HR 애플리케이션에서 사용자 목록을 .CSV 형식으로 내보내는 것이 좋습니다. 이 단계에서 사용자 일괄 가져오기를 하지 않아도 CSV 형식을 편하게 사용할 수 있습니다. Learning Manager에서 가져오기 프로세스를 시작하려면 .CSV 파일을 업로드하고 응용 프로그램 데이터 필드를 조직의 CSV 열에 매핑하십시오. 자세한 내용은 [일괄 가져오기 도움말](add-users-in-bulk.md)을 참조하십시오.

### FTP 커넥터 통합 {#ftpconnectorintegration}

조직에서 직원을 계속해서 추가하거나 제거해야 한다면 FTP 커넥터를 사용하여 사용자의 일괄 가져오기 자동화를 선택할 수 있습니다. 먼저 연결을 설정한 다음 CSV를 업로드하고 해당 Learning Manager 필드에 CSV 속성을 매핑할 수 있습니다. 자동 가져오기 프로세스를 예약하고 필요에 따라 동기화할 수도 있습니다. 자세한 내용은 [FTP 커넥터 도움말](../integration-admin/feature-summary/connectors.md#ftpconnector)을 참조하십시오.

### Salesforce 커넥터 통합 {#salesforceconnectorintegration}

조직에 Salesforce 계정이 있으면 Salesforce에서 자동으로 모든 사용자를 Learning Manager 응용 프로그램으로 한 번에 가져올 수 있습니다. 이 기능을 사용하려면 Learning Manager를 Salesforce 계정과 통합하고 데이터 동기화를 자동화하여 Salesforce 커넥터를 사용할 수 있습니다. 자동 사용자 가져오기를 정기적으로 수행하기 위해 자동 스케줄링 기능을 사용합니다. 또한, 매일 동기화를 수행할 수도 있습니다. 자세한 내용은 [Salesforce 커넥터 도움말](../integration-admin/feature-summary/connectors.md#sfconnector)을 참조하십시오.

### Adobe Connect 통합 {#adobeconnectintegration}

조직에서 Adobe Connect를 사용하면 Adobe Learning Manager 응용 프로그램과 통합하여 학습자에게 원활한 사용자 환경을 제공할 수 있습니다. 이러한 통합으로 학습자는 Adobe Connect에 다시 로그인할 필요 없이 Learning Manager 응용 프로그램에서 한 번의 클릭으로 바로 가상 교실에 접속할 수 있습니다. 이 기능을 사용하면 학습자는 Learning Manager 응용 프로그램 내에서 녹화된 가상 클래스 룸 세션을 사용할 수도 있습니다. 이러한 통합을 진행하려면 먼저 설정부터 통합합니다. 연결 URL 및 로그인 자격 증명을 제시하고 통합하려면&#x200B;**&#39;설정&#39; > &#39;Adobe Connect&#39; > &#39;지금 구성&#39;**&#x200B;을 사용합니다. 자세한 내용은 [Adobe Connect 통합 도움말](feature-summary/adobeconnect-integration.md)을 참조하십시오.

### Salesforce 앱 등록 {#salesforceappregistration}

기업 학습자는 Salesforce 계정에서 바로 Learning Manager 앱으로 원활히 접속할 수 있습니다. 학습자는 Salesforce 앱에서 강의, 학습 프로그램, 작업 지원 등 할당된 학습 콘텐트에 접속할 수 있습니다. 또한, 사용자는 Salesforce 안에 있는 Learning Manager 응용 프로그램에서 알림 및 공지사항에 접근할 수도 있습니다. 이 기능을 사용하려면 Learning Manager 응용 프로그램에서 Salesforce 추천 앱을 등록해야 합니다. 설치 및 사용 지침에 대한 자세한 내용은 [Salesforce 앱 도움말](../integration-admin/feature-summary/sfdc-app.md)을 참조하십시오.

## 2단계 - 사이트 구성(Learning Manager 책임자) {#phase2siteconfigurationcaptivateprimeadministrator}

조직에서 Learning Manager 응용 프로그램 관리자는 학습자를 위해 일부 기능을 사용하기 전에 먼저 기능을 구성하거나 설정해야 합니다.

### 브랜딩 {#branding}

조직은 Learning Manager 응용 프로그램에 회사 로고를 표시하고, URL에 자체 도메인을 가지며, 조직 이름을 표시하고, 조직 브랜드와 일치하는 색상 구성표를 표시할 수 있습니다. 조직은 Learning Manager에서 이러한 모든 기능을 사용할 수 있습니다. 사용자 정의 설정과 자체 브랜드 사용을 원하면 왼쪽 창에서 브랜드 섹션을 클릭합니다. 모든 옵션 옆에 있는 &#39;편집&#39;을 클릭하고 원하는 대로 변경합니다. 자세한 내용은 [브랜딩 및 테마 도움말](feature-summary/themes.md)을 참조하세요.

### 사용자/사용자 그룹 추가 {#addusersusergroups}

학습자가 학습 콘텐트의 주요 사용자이기 때문에 학습 관리 시스템에 사용자를 추가하는 것은 중요한 단계입니다. Learning Manager 응용 프로그램에서 사용자를 추가하고 역할을 부여합니다. 다음과 같은 방법으로 사용자를 추가할 수 있습니다.

#### 사용자 추가 (내부) {#addusersinternal}

* **단일 사용자** - 일괄로 단일 사용자들을 추가하기 전에 Learning Manager 응용 프로그램에 추가해 보면 일부 사용자에 대하여 테스트해 볼 수 있습니다. 또한 이 옵션은 사용자를 대량으로 가져온 후 필요에 따라 단일 사용자를 더 추가하려는 경우에 유용합니다.
* **자체 등록** - 관리자가 이 옵션을 사용하면 직원이 Learning Manager에 직접 등록할 수 있습니다.
* **일괄 가져오기**(CSV 업로드 사용) - 이 옵션을 사용하면 사용자를 Learning Manager 응용 프로그램으로 일괄 가져올 수 있습니다. 이 기능을 사용하려면 먼저 사용자 목록을 CSV 형식으로 준비해야 합니다.

#### 사용자 추가 (외부 프로필) {#addusersexternalprofiles}

* 외부 등록 - 이 옵션을 사용하면 외부 부서 구성원 또는 조직 외부 직원을 응용 프로그램에 등록할 수 있습니다.

#### 사용자 그룹 추가 {#addusergroups}

Learning Manager 응용 프로그램은 유사한 속성을 토대로 하여 기본 사용자 그룹을 생성합니다. 기본 그룹 외에도 직함, 위치 등의 변수를 기반으로 사용자 정의 사용자 그룹을 생성하여 이러한 그룹을 게임화 및 보고서에 활용할 수 있습니다. 자세한 내용은 [사용자 추가 도움말](feature-summary/add-users-user-groups.md)을 참조하십시오.

### 역할 부여 {#assignroles}

사용자가 Learning Manager에 추가되면 관리자는 조직 요구 사항에 따라 사용자에게 작성자, 관리자 또는 통합 책임자 역할 부여를 시작할 수 있습니다. 사용자에게 역할을 할당하려면 관리자 로그인에서 왼쪽 창에서 **[!UICONTROL 사용자]**&#x200B;를 클릭하고 각 사용자 이름에 대한 확인란을 선택한 다음 **[!UICONTROL 작업]** 드롭다운을 클릭하여 할당할 역할을 선택합니다. 관리자 역할은 CSV 파일에서 조직이 언급한 역할/권한에 따라 Adobe Learning Manager이 사용자에게 할당합니다. 사용자 편집 워크플로우를 사용하여 관리자 역할을 변경할 수도 있습니다. 자세한 내용은 [사용자 추가 도움말](feature-summary/add-users-user-groups.md)을 참조하십시오.

### 알림 템플릿 {#notificationtemplates}

알림은 학습 관리 시스템의 사용자가 자신의 상태를 알거나 이벤트/활동 정보를 받을 때 유용합니다. 강의를 만들고 기능을 구성하거나 강의를 이용하는 동안 사용자는 학습자, 책임자, 관리자, 작성자에게 알림을 보내는 여러 이벤트를 경험합니다. Learning Manager는 응용 프로그램에 다양한 전자 메일 알림 템플릿을 제공합니다. 관리자는 조직 요구 사항에 따라 이를 조정할 수 있습니다. 전자 메일 알림을 생성하려면 왼쪽 창에서 **전자 메일 템플릿**&#x200B;을 선택하고 템플릿 이름을 클릭합니다. 자세한 내용은 [전자 메일 도움말](feature-summary/email-templates.md)을 참조하십시오.

**전자 메일 알림 해제(권장)**

기본적으로 Learning Manager 응용 프로그램에서는 알림이 활성화되어 있습니다. 조직의 직원들에게 알림을 보내길 원하지 않으면 이 단계에서 알림을 해제할 수 있습니다.

### 배지 {#badges}

배지는 직원이 강의 완료 시 획득할 수 있는 성과에 대한 척도입니다. 전 세계의 교수가 특정 스킬 또는 학습 목표의 표상으로 배지를 사용합니다. 배지 컬렉션을 만들어 학습 콘텐트를 완료하면 학습자에게 배지를 수여할 수 있습니다. 배지 제작을 시작하려면 왼쪽 창에서 **[!UICONTROL 배지]** 기능을 클릭합니다. 자세한 내용은 [배지 도움말](feature-summary/badges.md)을 참조하십시오.

### 피드백 {#feedback}

LMS에서 피드백은 학습자의 학습 진척도를 측정하고 학습 콘텐트의 품질을 확인하기 위한 중요한 요소입니다. Learning Manager는 사용자에게 두 가지의 피드백을 제공합니다.

* L1 피드백은 강의를 완료하면 학습자에게 제공하는 피드백입니다.
* L3 피드백은 학습자의 행동과 일상적인 활동에 대한 강의 영향을 기반으로 관리자가 학습자에게 제공하는 피드백입니다.

조직에서 선택에 따라 이 기능을 사용할 수 있습니다. 관리자는 조직의 요구 사항에 따라 피드백 템플릿을 조정할 수 있습니다. 이 기능을 사용하려면 책임자가 강의 인스턴스 레벨에서 L1 및 L3 피드백 옵션을 허용해야 합니다. 피드백을 구성하려면 강의를 선택하고 왼쪽 창에서 인스턴스 기본값을 클릭한 다음 피드백 옵션을 사용하도록 설정합니다. 자세한 내용은 [피드백 도움말](feature-summary/courses.md)을 참조하십시오.

### 게임화 {#gamification}

조직이 해결해야 하는 과제는 학습자가 콘텐트를 이용하는 동안 참여를 유도하는 것입니다. 게임화는 게임 기술을 통해 학습자가 목표를 달성하도록 유도하고 동기를 부여하여 강의 완료도를 높입니다. 학습자는 동료와 경쟁하며 다양한 학습 활동의 점수를 획득하고 브론즈, 실버, 골드, 플래티넘 레벨을 달성할 수 있습니다. 관리자는 각 게임화 작업을 활성화/비활성화로 설정하고 작업에 대한 점수 할당을 조정할 수 있습니다. Learning Manager는 일부 기본 설정과 함께 게임화 기능을 제공합니다. 한동안은 기본 설정과 함께 이 기능을 사용할 수 있고 그리고 나서 조정이 가능합니다. 이 기능에 대한 설정을 구성/변경하는 것을 선택할 수 있습니다. 조직에 서브젝트 전문가가 있으면 사용하기 전에 설정을 구성하는 것이 좋습니다. 자세한 내용은 [게임화 도움말](feature-summary/gamification.md)을 참조하십시오.

### 학습 객체 생성 {#createlearningobjects}

이 단계는 세 가지 트랙(트랙1, 트랙 2 및 트랙 3)이 모두 모여 다음 행동을 취하는 논리 단계입니다.

이 시점에서 모듈 및 강의를 생성하면 인증, 학습 프로그램, 작업 지원, 그리고 더 나아간 상위 수준의 학습 객체 생성을 시작할 수 있습니다. 학습 개체 생성을 시작하기 전에 일부 사용자를 Learning Manager 응용 프로그램에 미리 추가하고 일부 모듈과 강의를 생성했는지 확인합니다.

#### 인증 생성 {#createcertifications}

인증은 학습 내용 완료나 일회 또는 반복적인 기간 동안 학습자의 성취를 증명합니다. 강의 완료 후 학습자에게 인증을 제공하여 학습 콘텐트에 학습자 등록률을 높일 수 있습니다. 관리자는 내부적으로 호스팅하거나 제3자가 수행하는 인증 프로그램을 생성할 수 있습니다. 내부 인증에서는 학습자가 인증을 받기 위해 이수해야 하는 강의를 정해야 합니다. 인증을 생성하기 전에 계정에 사용 가능한 강의가 몇 가지 있는지 확인합니다. 인증을 만드는 방법에 대한 자세한 내용은 [인증 도움말](feature-summary/certifications.md)을 참조하십시오.

#### 작업 지원 생성 {#createjobaids}

작업 지원은 등록 또는 완료 조건 없이 학습자가 액세스할 수 있는 교육 콘텐츠 저장소입니다. 학습자는 일하는 동안 이러한 작업 지원을 참조하여 조직 내 활동이나 작업 수행을 지원받을 수 있습니다. 강의 생성 부분으로 작업 지원 장치를 사용해야 하는 것은 아니지만, Learning Manager 팀은 조직 모범 사례로 작업 지원 생성을 권장합니다. 작업 지원을 만드는 방법에 대한 자세한 내용은 [작업 지원 도움말](../authors/feature-summary/job-aids.md)을 참조하십시오.

#### 학습 프로그램 생성 {#createlearningprograms}

학습 프로그램은 특정 학습자가 목표를 이룰 수 있도록 맞춰 설계된 강의입니다. 학습 프로그램을 만들기 전에 이미 일부 강의를 만들었는지 또는 계정에서 사용할 수 있는 기존 강의가 있는지 확인하십시오.  조직에서 이 기능을 사용하는 것은 선택 사항이지만, Learning Manager 팀에서는 이 기능을 사용하여 직원에게 집중된 학습을 주입할 것을 권장합니다. 학습 프로그램 사용을 시작하려면 왼쪽 창에서 &#39;학습 프로그램&#39;을 클릭하고 카탈로그에서 &#39;강의 모음&#39;을 선택하고 게시합니다. 자세한 지침은 [학습 프로그램 도움말](feature-summary/learning-programs.md)을 참조하십시오.

### 카탈로그 생성 {#createcatalogs}

조직의 카탈로그를 사용하여 목표 콘텐트를 만들거나 해당 학습자에 맞게 분류된 콘텐트를 생성할 수 있습니다. Learning Manager의 카탈로그는 강의, 인증 또는 학습 프로그램과 같은 학습 객체가 모여 있습니다. 카탈로그를 만들면서 선택할 학습 객체를 고를 수 있습니다. 카탈로그를 만들기 전에 강의, 인증 또는 학습 프로그램을 미리 작성했는지 확인합니다. 내부 또는 외부 사용자 그룹에 할당하고 싶을 경우 학습 객체 세트로 카탈로그를 사용자 정의 생성할 수 있습니다. 카탈로그에 대한 자세한 내용은 [카탈로그 도움말](feature-summary/catalogs.md)을 참조하십시오.

### 전자 메일 알림/사용자 액세스 켜기 {#turnonemailnotificationsuseraccess}

이 단계에서 애플리케이션의 사용자에게 이메일 알림을 켜고 사용자 액세스를 활성화할 수도 있습니다.

## 3단계 - 강의 작성(작성자) {#phase3authoringcoursesauthor}

조직의 콘텐츠 개발자 또는 작성자가 학습 콘텐츠를 만들 수 있습니다. Learning Manager 응용 프로그램에서 모든 학습 콘텐츠의 기초가 되는 모듈과 강의는 반드시 만들어야 합니다.

### 모듈 생성 {#createmodules}

모듈은 Learning Manager 응용 프로그램의 기본 구성 요소입니다. 학습 내용을 구성하려면 작성자로 모듈 생성을 시작합니다. Learning Manager를 사용하면 강의실, 자가 학습, 활동 및 가상 강의실 모듈과 같은 네 가지 유형의 강의 모듈 중 하나를 선택할 수 있습니다. 조직의 요구 사항에 가장 적합한 강의 모듈 유형에 대한 자세한 내용은 [모듈 도움말](../authors/how-to-choose-modules.md)을 참조하십시오.

### 강의 생성 {#createcourses}

관리자는 Learning Manager 응용 프로그램에서 강의를 만드는 작성자 역할을 수행할 수 있습니다. Learning Manager 응용 프로그램에서 강의는 학습자에게 할당할 모듈 집합의 기본 단위입니다. 학습자는 강의를 이용합니다. 이전에 생성한 모듈을 선택하여 강의 생성을 시작하고, 학습자들이 강의에서 달성하길 원하는 스킬을 연결하고, 레벨, 크레딧 및 배지를 강의에 연동하고, 선택을 기반으로 작업 지원, 전제 조건 및 리소스를 선택하고, 강의를 게시할 수 있습니다. 강의 인스턴스를 여러 개 생성하여 여러 시간대에 있는 다양한 학습자에게 강의 인스턴스를 부여할 수도 있습니다. 강의를 만드는 방법에 대한 자세한 내용은 [강의 도움말](../authors/feature-summary/courses.md)을 참조하십시오.

## 4단계 - LMS 관리(Learning Manager 책임자) {#phase4managingyourlmscaptivateprimeadministrator}

### 공지사항 {#announcements}

공지는 모든 학습자 계정에 중요한 정보를 한 번에 전송할 때 유용합니다. Learning Manager 응용 프로그램에서는 사용자가 정의한 사용자 그룹과 학습 개체에 멀티미디어 메시지(텍스트, 이미지 또는 비디오)로 공지를 보낼 수 있습니다. 공지사항을 즉시 보내거나 특정 날짜에 자동으로 발송되도록 예약할 수 있습니다. Learning Manager 응용 프로그램에서 이 기능을 사용하려면 왼쪽 창에서 &#39;공지&#39;를 선택하고 &#39;추가&#39;를 클릭하여 공지를 만듭니다. 자세한 내용은 [공지 도움말](feature-summary/announcements.md)을 참조하십시오.

### 사용자 등록 {#userenrollment}

사용자 등록은 LMS 응용 프로그램에서 중요한 단계입니다. 이 단계에서는 Learning Manager 응용 프로그램의 다양한 학습 객체에 학습자 등록을 시작할 수 있습니다. 학습 계획을 사용하여 수동 또는 자동으로 학습자를 등록할 수 있습니다.

#### 매뉴얼 등록 {#manualenrollment}

* **자체 등록 -** 학습 개체를 만드는 동안 이 옵션을 활성화하면 학습자가 학습 개체에 직접 등록할 수 있습니다.
* **관리자 승인** - 강의를 만드는 동안 등록 유형에서 이 옵션을 선택하면 관리자가 학습자의 등록을 승인해야 합니다.
* **관리자 지정** - 강의를 만드는 동안 이 등록 유형을 선택하면 관리자가 해당 강의를 지정해야 합니다.
* **관리자 등록** - 이 경우 관리자는 조직의 요구 사항에 따라 일부 학습자를 등록할 수 있습니다.

자세한 내용은 [사용자 등록 도움말](feature-summary/courses.md)내용을 참조하십시오.

학습자를 수동으로 학습 객체에 등록하는 데는 다음 네 가지 방법이 있습니다.

### 자동 등록 {#automatedenrollment}

학습 계획을 사용하여 학습자의 학습 객체 등록을 자동화합니다.

#### 학습 계획(트리거 기반) {#learningplansbasedontriggers}

특정 이벤트에 따라 학습자에게 강의, 학습 프로그램 또는 인증을 자동으로 부여하기 위해 학습 계획을 사용할 수 있습니다. For example,

* 새 사용자 추가
* 사용자 그룹 변경
* 학습 객체 완료
* 스킬 습득 완료

자세한 내용은 [학습 계획 도움말](feature-summary/learning-plans.md) 콘텐츠를 참조하십시오.

### 보고서 만들기 {#createreports}

이 단계에서는 다양한 유형의 보고서를 생성하고 관리할 수 있습니다.

Adobe Learning Manager를 사용하면 학습자 활동을 추적, 모니터링 및 제어하는 다양한 보고서를 생성할 수 있습니다. 다음 세 가지 유형의 보고 기능을 사용하여 보고서를 생성할 수 있습니다.

* **보고서 대시보드** - 요약 보고서를 만들어 사용자 그룹, 효과, 학습자의 강의 시간 등과 같은 다양한 범주를 기반으로 학습자의 학습 콘텐츠 소비에 대한 실행 가능한 통찰력을 얻습니다.
* **학습자 성적 증명서** - 등록일로부터 현재까지 학습자 전체 기록을 사용하여 학습자 중심 보고서를 만듭니다.
* **강의 보고서** - 개별 강의에서 학습자의 강의 사용 통계를 만듭니다. 강의 수준에서 퀴즈 보고서를 생성할 수도 있습니다.

보고서 생성 프로세스에 대한 자세한 내용은 [보고서 도움말](feature-summary/reports.md)을 참고하십시오.

Learning Manager 응용 프로그램 기능 사용과 관련된 다른 정보는 각 역할에 따른 [Learning Manager 도움말](../topics.md)을 참조하십시오.
