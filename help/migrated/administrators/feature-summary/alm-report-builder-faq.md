# Report Builder - 자주 묻는 질문


**템플릿과 보고서의 차이점은 무엇입니까?**

템플릿은 Adobe Learning Manager에서 제공하는 사전 구축된 보고서 구성입니다. 일반적인 사용 사례용으로 설계되었으며 즉시 다운로드할 수 있고 읽기 전용입니다. 편집할 수 없습니다. 보고서는 사용자가 직접 저장한 구성입니다. 처음부터 새로 만들거나 템플릿을 복제하고 사본을 편집하여 만들 수 있습니다. 보고서는 **보고서** 탭에 나타나고 템플릿은 **템플릿** 탭에 나타납니다.

**템플릿을 직접 편집할 수 있습니까?**

아니요. 템플릿은 읽기 전용입니다. 템플릿을 사용자 정의하려면 **복제**&#x200B;를 선택하여 편집 가능한 복사본을 만듭니다. 변경 내용은 **보고서** 탭에 새 보고서로 저장되며 원본 템플릿에는 영향을 주지 않습니다.

중복 행은 데이터의 필드가 기본 레코드와 일대다 관계에 있을 때 나타납니다. 일반적인 원인으로는 여러 강사가 있는 세션(세션당 강사당 한 행) 및 다중 작성자가 있는 학습 객체가 있습니다. 이 문제를 해결하려면 **강사 이름** 또는 **작성자 이름**&#x200B;과 같은 여러 값이 있는 필드를 열로 추가하십시오.

**동일한 보고서의 두 다운로드 간에 행 순서가 변경되는 이유는 무엇입니까?**

보고서 생성은 분산 쿼리 시스템을 사용합니다. 명시적 정렬이 없으면 행 순서는 실행 간에 차이가 있을 수 있는 결과를 먼저 반환하는 쿼리 노드에 따라 달라집니다. 다운로드 전체에 걸쳐 출력을 일관되게 유지하려면 보고서에 정렬을 하나 이상 적용하십시오.

**날짜가 UTC로 표시되는 이유는 무엇입니까?**

Report Builder은 이번 릴리스에서 날짜 값을 UTC로 반환합니다. 계정에 구성된 시간대는 향후 릴리스의 날짜 필드에 적용됩니다. 날짜 기반 데이터를 분석할 때 계정의 기본 시간대와 관련된 UTC 오프셋을 고려합니다.

**새 등록 또는 완료 데이터가 나타나는 데 얼마나 걸립니까?**

Report Builder은 Adobe Learning Manager 데이터베이스에서 가져오며, 시스템에서 이벤트가 발생할 때 최대 지연 시간은 약 15분입니다. 방금 등록이나 완료를 기록했는데 보고서에 나타나지 않으면 15분 이상 기다린 후 다시 다운로드하십시오.

**보고서의 행 또는 열에 제한이 있습니까?**

보고서는 약 100만 행으로 제한됩니다. 이 릴리스의 열 수에는 제한이 없습니다. 보고서에 100만 개가 넘는 행이 필요한 경우 필터를 적용하여 범위를 좁힙니다.

**API 또는 자동화를 통해 Report Builder 데이터를 가져올 수 있습니까?**

Report Builder 보고서에 대한 자동 API 액세스는 향후 릴리스에서 계획되어 있습니다. 현재 릴리스에서는 Report Builder UI를 통해 보고서가 수동으로 다운로드되거나 구독 기능을 통해 예약된 방식으로 수신됩니다.
