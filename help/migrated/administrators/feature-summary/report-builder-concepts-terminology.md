# Report Builder - 개념 및 용어

## 템플릿 및 보고서

**템플릿**&#x200B;은(는) Adobe Learning Manager에서 제공하는 사전 구성된 보고서 구성입니다. 일반적인 사용 사례, 등록 및 완료 추적, 규정 준수 보고, 강사 성과 등을 위해 설계되었으며 즉시 다운로드할 준비가 되어 있습니다. 템플릿은 읽기 전용이므로 편집하거나 덮어쓸 수 없습니다.

![](assets/report-builder-0002.png)

**보고서**&#x200B;는 사용자가 저장한 구성입니다. 처음부터 보고서를 작성하거나 템플릿을 복제하고 사본을 편집하여 보고서를 작성할 수 있습니다. 템플릿을 복제하면 복사본은 **보고서** 탭에서 보고서가 됩니다. 템플릿과 보고서 모두 Report Builder에 표시되지만 별도의 탭에 표시됩니다.

## 데이터 세트

데이터 집합은 Report Builder 내의 관련 열의 명명된 그룹입니다. 보고서에 열을 추가하면 이러한 데이터 집합에서 열이 선택됩니다. 각 데이터 세트를 학습 데이터의 한 측면에 대한 정보 테이블로 간주합니다.

다음은 Report Builder에서 사용할 수 있는 데이터 집합 샘플입니다.

* **사용자**: 활성 필드를 포함한 학습자 프로필 데이터
* **성적 증명서**: 등록 및 완료 기록
* **학습 개체**: 과정, 학습 경로 및 인증 데이터
* **학습 개체 인스턴스**: 인스턴스 수준 세부 정보
* **카탈로그**: 카탈로그 및 카탈로그 레이블 데이터
* **사용자 그룹**: 사용자 그룹 구성원 자격 및 계층 구조
* **모듈**: e-러닝 모듈 세부 정보를 포함한 강의실 및 가상 세션 데이터

>[!NOTE]
>
>데이터 세트는 선택적으로 조인할 수 있습니다. 단일 보고서에서 모든 조합을 사용할 수 있는 것은 아닙니다.

## 열 및 추가 버튼

추가한 각 열은 보고서 캔버스에 행으로 표시되고 다운로드한 파일의 열이 됩니다. 동일한 열을 두 번 이상 추가할 수 있습니다. 이는 동일한 필드에서 두 가지 다른 값을 측정하려는 경우에 유용합니다. 예를 들어 **상태** 열을 두 번 추가할 수 있습니다. 한 번은 등록 수를 계산하고 한 번은 진행 중인 학습자 수를 합산하여 계산합니다.

![](assets/report-builder-0003.png)

별칭을 입력하여 열의 이름을 바꿀 수도 있습니다. 별칭은 다운로드한 보고서에서 열 머리글로 나타납니다.

## 그룹화 기준 및 합산

그룹화 기준 은 개별 행을 표시하지 않고 선택한 필드를 기준으로 데이터를 요약합니다. 예를 들어, 강사 이름별로 그룹화하면 등록당 하나의 행이 아니라 강사당 하나의 행이 제공됩니다.

그룹화 기준은 표준 데이터베이스 동작을 따릅니다. 한 열에 그룹화 기준을 적용한 경우 보고서의 다른 모든 열에 집계 함수가 적용되어야 합니다. 개별 행 데이터와 그룹화된 데이터를 혼합할 수 없습니다. 사용 가능한 집계 함수는 다음과 같습니다.

* **개수**: 총 행 수
* **다음 경우 개수**: 필드가 지정한 값과 일치하는 행 수
* **합계**: 총 숫자 필드
* **분**: 숫자 필드의 가장 낮은 값
* **최대**: 숫자 필드의 가장 높은 값
* **평균**: 숫자 필드의 평균값

Excel에서 피벗 테이블을 사용한 경우 그룹화는 열 수준에서 동일한 방식으로 작동합니다.

## 필터

필터는 보고서에 표시할 행을 제한합니다. 여러 필터를 적용하고 AND 또는 OR 논리와 결합할 수 있습니다.

필터 연산자는 열의 데이터 형식에 따라 다릅니다.

* **문자열 필드**: 포함, 같음, 다음으로 시작(인식된 값에 사용할 수 있는 형식 미리 검색)
* **숫자 필드**: 보다 큼, 보다 작음, 같음, 사이
* **날짜 필드**: 같음, 이전, 이후, 사이 및 상대 범위(예: 최근 90일)
* **열거형(목록) 필드**: 있음, 없음(다중 값 선택)

## AND/OR 논리 및 중첩 필터 그룹

다중 필터는 기본적으로 AND 논리로 설정되며 행이 나타나려면 모든 조건이 true여야 합니다. 두 필터 간에 연산자를 OR로 전환할 수 있습니다. 괄호를 만드는 **그룹으로 추가**&#x200B;를 사용하여 필터를 그룹화할 수도 있습니다. 그룹 내의 필터는 외부의 필터와 결합되기 전에 함께 평가됩니다.

이렇게 하면 다음과 같은 조건을 구축할 수 있습니다. (카탈로그 = 안전 또는 카탈로그 = 위생) 완료 일자는 최근 90일 동안입니다.

복잡한 다중 수준 논리를 지원하기 위해 다른 그룹 내에 그룹을 중첩할 수 있습니다.

## 정렬

하나 이상의 열을 정렬할 수 있습니다. 정렬되는 첫 번째 열은 기본 정렬입니다. 추가 정렬은 기본 열의 연결 내에 적용됩니다.

일관된 출력이 필요한 경우 항상 하나 이상의 정렬을 적용합니다. 보고서 생성은 분산 시스템에서 실행되므로 정렬이 적용되지 않는 한 동일한 보고서의 두 다운로드 간에 행 순서가 보장되지 않습니다.

## 추세 데이터와 스냅샷 데이터 비교

월정액 또는 주정액과 같이 추세 집계기를 사용하는 모든 보고서에는 날짜별로 그룹화된 현재 스냅샷 데이터가 반영됩니다. 과거 각 날짜의 데이터 기록 상태를 반영하지 않습니다.

예를 들어, 월별로 그룹화된 등록 추세는 현재 존재하는 등록 수를 표시하며, 이 등록 수가 생성된 달에 걸쳐 분산됩니다. 나중에 사용자 그룹을 등록 취소하거나 변경한 학습자는 해당되지 않습니다. 이러한 변경 사항은 지난 달에 소급하여 적용되지 않습니다.

## 삭제된 학습자 및 활성 필드

Report Builder은 삭제된 학습자를 보고서에 포함하거나 활성 필드 값을 검색할 수 있도록 지원합니다.

사용자 데이터 집합의 삭제 날짜 열을 사용하여 보고서를 만듭니다.

## 모범 사례

* 보고서를 처음부터 작성하기 전에 사용 가능한 데이터 집합 참조를 읽으십시오. 필요한 필드가 포함된 데이터 세트를 알면 구성 시간을 크게 절약할 수 있습니다.
* 예약된 보고서를 구독하기 전에 정렬을 적용합니다. 이렇게 하면 모든 전달 시 행 순서가 일관되게 유지됩니다.
* 예기치 않은 중복 행이 표시되는 경우 보고서에 여러 강사가 있는 세션에 대한 강사 이름과 같이 행당 여러 값을 가질 수 있는 필드가 포함되어 있는지 확인합니다.
