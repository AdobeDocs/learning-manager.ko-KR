# Adobe Learning Manager의 고급 설정

## 카탈로그 레이블

Adobe Learning Manager의 카탈로그 레이블은 학습 객체(강의, 인증, 학습 경로 등)에 태그를 지정하는 데 사용됩니다. 특정 필드 및 값 포함. 사용자와 작성자는 이러한 레이블을 사용하여 컨텐츠를 효과적으로 분류 및 구성하여 필터링, 추적 및 보고 기능을 향상시킬 수 있습니다.

자세한 내용은 [Adobe Learning Manager의 카탈로그 레이블](/help/migrated/administrators/feature-summary/catalog-labels.md)을 참조하세요.


>[!NOTE]
>
>* 필수 레이블: 강의 생성 시 작성자의 카탈로그 레이블을 필수 항목으로 지정할 수 있습니다.
>* 작성자 작업 과정: 작성자는 강의를 만들거나 편집하는 동안 준수 레이블을 추가하여 적절한 분류를 해야 합니다.

## 콘텐츠 폴더

Adobe Learning Manager의 콘텐츠 폴더는 작성자가 콘텐츠 라이브러리의 콘텐츠를 보고 액세스할 수 있는 사용자를 제어합니다. 계층적 콘텐츠 폴더를 사용하면 관리자가 대용량 콘텐츠 라이브러리를 최대 세 가지 수준의 중첩된 비공개 폴더로 구성하여 조직 전체에서 콘텐츠를 보다 쉽게 찾고, 관리하고, 재사용할 수 있습니다.

### 콘텐츠 폴더란?

콘텐츠 폴더는 관련 콘텐츠를 그룹화하고 해당 폴더에 액세스할 수 있는 사용자를 결정하는 컨테이너입니다. Adobe Learning Manager의 모든 콘텐츠 파일은 항상 하나 이상의 폴더에 속합니다.

콘텐츠 폴더에는 두 가지 유형이 있습니다.

**공용 폴더**- 기본적으로 모든 계정에 있습니다. 공용 폴더에는 다음과 같은 속성이 있습니다.

* 계정의 모든 작성자는 공개 폴더의 콘텐츠에 액세스할 수 있습니다.
* 공개 폴더의 콘텐츠는 비공개 폴더에 있을 수 없습니다. 그 반대도 마찬가지입니다. 비공개 폴더의 콘텐츠는 공개 폴더에 있을 수 없습니다.
* 공용 폴더는 역할 기반 액세스 구성에 속하지 않습니다. 사용자 정의 역할을 특정 개인 폴더로 제한해도 공용 폴더에 대한 액세스는 제한되지 않습니다.

**개인 폴더**- 관리자가 만들었습니다. 개인 폴더는 3단계 계층을 지원하며 역할 구성을 통해 액세스가 제어됩니다.

**폴더 계층 구조 수준 이해**

개인 콘텐츠 폴더는 최대 세 가지 수준의 중첩을 지원합니다.

* **수준 1 폴더** - 콘텐츠 라이브러리의 루트에 있는 최상위 폴더

* **수준 2 폴더**- 수준 1 폴더 안에 중첩된 하위 폴더

* **수준 3 폴더**- 수준 2 폴더 안에 중첩된 하위 폴더

따라서 수천 개의 파일을 단순 목록으로 관리하는 대신 주제 영역, 제공 유형, 대상 그룹 또는 팀별로 실제 컨텐츠 구성을 미러링할 수 있는 유연성을 제공합니다.

관리자만 모든 수준에서 폴더를 만들거나 편집하거나 삭제할 수 있습니다. 작성자와 사용자 정의 사용자는 계층과 상호 작용하지만 수정할 수 없습니다.

### 폴더 이름 지정 규칙

폴더 이름은 같은 상위 폴더 아래의 같은 수준 내에서 고유해야 합니다. 특히,

| **시나리오** | **허용됩니까?** |
|----------------------------------------------------------------------------------------------|--------------------------|
| 이름이 같은 두 개의 수준 1 폴더 | 아니오 |
| 동일한 이름의 동일한 레벨 1 폴더 아래에 있는 두 개의 레벨 2 폴더 | 아니오 |
| 동일한 이름의 서로 다른 수준 1 폴더 아래에 있는 두 개의 수준 2 폴더 | 예 |
| 이름이 같은 레벨 2 폴더와 레벨 3 폴더 | 예. 레벨은 고유합니다. |
| 동일한 이름의 동일한 레벨 2 폴더 아래에 있는 레벨 3 폴더와 다른 레벨 3 폴더 | 아니오 |


### 폴더 경로가 표시되는 방식

컨텐츠 라이브러리에는 각 컨텐츠 파일의 전체 폴더 경로가 표시됩니다. 예를 들어 **교육 프로그램** > **온보딩** > **SCORM 에셋**&#x200B;을 선택합니다. 이 경로는 콘텐츠의 전체 위치를 보여 줍니다.

파일이 두 개 이상의 폴더에 있는 경우 모든 경로가 쉼표로 구분되어 표시됩니다. 경로가 길면 처음부터 줄임표(...)로 잘리고 가장 깊은 폴더 이름이 항상 표시됩니다.

### 폴더에 대한 역할 기반 액세스

개인 폴더에 대한 액세스 권한은 **수준 1에만**&#x200B;에 할당됩니다. 사용자 정의 역할에 레벨 1 폴더에 대한 액세스 권한이 부여되면 해당 액세스 권한은 해당 폴더 내의 모든 레벨 2 및 레벨 3 하위 폴더로 자동으로 캐스케이드됩니다. 하위 폴더 레벨에서는 독립적으로 액세스 권한을 부여할 수 있는 옵션이 없습니다.

다음 표에서는 각 역할이 폴더 계층 구조에서 수행할 수 있는 작업에 대해 설명합니다.

| **역할** | **수행할 수 있는 작업** |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|
| 책임자 | 수준 1, 수준 2 및 수준 3 개인 폴더를 만들고 이름을 바꾸고 삭제합니다. 사용자 정의 역할에 대해 수준 1 폴더 액세스를 구성합니다. |
| 사용자 정의 관리자 | 할당된 권한에 따라 액세스 가능한 레벨 1 분기 내의 폴더 관리 |
| 작성자 | 폴더를 검색하고, 폴더별로 콘텐츠를 필터링하고, 폴더에 콘텐츠를 추가하고, 폴더 간에 콘텐츠를 복사 및 이동하고, 강의에 모듈을 추가할 때 콘텐츠를 선택합니다. |
| 사용자 정의 작성자 | 작성자와 동일하지만, 할당된 수준 1 권한을 통해 액세스할 수 있는 폴더로 제한됩니다. |

### 폴더 구조 제한

| **제한** | **값** |
|---------------------------------------|-----------|
| 계정당 레벨 1 폴더 | 제한 없음 |
| 수준 1 폴더당 수준 2 하위 폴더 | 25 |
| 레벨 2 폴더당 레벨 3 하위 폴더 | 25 |
| 최대 폴더 깊이 | 레벨 |


### 폴더 선택 비헤이비어

예를 들어, 필터링하거나 삭제할 때 폴더를 선택하면 선택 내용이 다음과 같이 계층 구조를 통해 캐스케이드됩니다.

* **수준 1 폴더**&#x200B;를 선택하면 그 아래에 있는 모든 수준 2 및 수준 3 폴더가 자동으로 선택됩니다.

* **수준 2 폴더**&#x200B;를 선택하면 그 아래에 있는 모든 수준 3 폴더가 자동으로 선택됩니다. 동일한 레벨 1 폴더 아래의 다른 레벨 2 폴더는 선택되지 않습니다.

* **수준 3 폴더**&#x200B;를 선택하면 해당 폴더만 선택됩니다. 다른 폴더는 선택되지 않았습니다.

>[!NOTE]
>
>마스터를 선택하지 않고 하위 폴더를 선택하면 상위 폴더에 부분 선택 표시기나 혼합 선택 표시기가 표시되지 않습니다. 이것은 의도적인 것이다. 상위 폴더에는 하위 폴더뿐만 아니라 콘텐츠 자체도 포함될 수 있기 때문입니다. 상위 폴더를 선택하면 &quot;이 폴더의 모든 내용과 그 아래의 모든 내용을 포함합니다.&quot;가 됩니다. 부분 표시기는 상위 폴더의 자체 콘텐츠가 부분적으로 포함되어 있음을 암시하므로 오해의 소지가 있을 수 있습니다. 특정 하위 폴더로만 필터링하려면 해당 하위 폴더를 직접 선택합니다. 상위 폴더 및 해당 하위 폴더의 모든 내용을 보려면 상위 폴더를 선택합니다.

### 계층적 폴더 구조를 사용하는 경우

계층형 컨텐츠 폴더는 조직에서 여러 컨텐츠 파일을 관리하고 이러한 파일에 대한 액세스를 탐색, 재사용 및 제어할 수 있는 구조적 방법이 필요한 경우 특히 유용합니다.

일반적인 시나리오는 다음과 같습니다.

* **대용량 콘텐츠 라이브러리**: 수천 개의 콘텐츠 파일이 있는 경우 3단계 계층 구조를 통해 작성자는 단순 목록을 스크롤하지 않고 필요한 항목을 직접 탐색할 수 있습니다.

* **여러 팀 또는 프로젝트**: 수준 1 폴더는 팀 또는 프로젝트 영역을 분리할 수 있고, 수준 2 폴더는 배달 유형별로 구성할 수 있으며, 수준 3 폴더는 개인 자산을 보유할 수 있습니다.

* **역할 기반 콘텐츠 분리**: 작성자 팀이 자신의 작업과 관련된 콘텐츠에만 액세스해야 하는 경우 수준 1 폴더 액세스 할당은 각 팀의 콘텐츠를 비공개로 유지합니다.

### 계층적 콘텐츠 폴더의 실제 사용 사례

**사용 사례 1- 관할 구역 관련 콘텐츠 준수 교육**

글로벌 조직은 여러 지역에 걸쳐 필수 규정 준수 교육을 실행합니다. 각 지역에는 모든 사람에게 적용되는 핵심 모듈과 데이터 개인정보 보호 규정, 현지 노동법, 재정 공개 요구 사항 등 국가나 지역에 따라 다른 관할 지역 관련 법률 추가 사항이 있습니다.

계층 폴더가 없으면 모든 규정 준수 에셋이 단순 목록에 포함되므로 지역 콘텐츠 팀이 어떤 파일이 어떤 프로그램 또는 관할권에 속하는지 알기 어렵습니다.

3단계 구조:

* 1단계: 규정 준수 교육

* 레벨 2: EMEA/APAC/미주(리전당 하나의 하위 폴더)

* 수준 3: 지역별 특정 모듈 또는 에셋(개인 정보 보호 규정 PDF, 로컬 정책 데크, 평가 파일)

지역 작성자 팀에는 레벨 1 또는 레벨 2 분기에만 액세스할 수 있습니다. 다른 지역의 콘텐츠를 보거나 실수로 수정하지 않고도 해당 관할권과 관련된 에셋만 찾아서 업데이트하고 재사용할 수 있습니다.

**사용 사례 2- 역할이 많은 대규모 온보딩 프로그램**

조직은 개별 기여자, 관리자, 계약자, 기술 전문가 등 여러 개별 역할에 걸쳐 연간 수천 명의 직원을 고용합니다. 각 역할에는 공유 기본 콘텐츠 및 역할별 모듈이 있는 자체 온보딩 트랙이 있습니다.

3단계 구조:

* 수준 1: 온보딩

* 레벨 2: 역할(개인 콘텐츠 작가/관리자/계약자/기술 전문가)

* 레벨 3: 모듈 유형(SCORM 패키지/ILT 데크/활동 가이드/평가)

각 역할의 강의를 작성하는 작성자는 레벨 2로 바로 이동하여 해당 트랙에 대한 정확한 파일을 찾습니다. 하나의 모듈이 회사 가치 비디오와 같은 여러 역할에서 재사용되는 경우 중복을 만들지 않고도 여러 폴더에 복사되거나 링크될 수 있습니다. 콘텐츠는 단일 소스이지만 모든 관련 분기에 나타납니다.

**사용 사례 3- 여러 콘텐츠 팀이 있는 대용량 기술 라이브러리**

기술 회사는 제품 라인, 클라우드 인프라, 개발자 도구, 보안 및 데이터 엔지니어링 전반에 걸쳐 수천 개의 콘텐츠 파일로 구성된 내부 스킬 교육 라이브러리를 유지 관리합니다. 여러 작성자 팀이 참여하며 각 팀은 하나의 제품 영역을 담당합니다. 강의 모듈은 강의당 40~60개의 파일을 실행할 수 있습니다.

계층 구조를 지정하지 않으면 수천 개의 모든 파일이 소수의 최상위 폴더에 배치되며 서로 다른 팀의 작성자가 파일 버전을 잘못 선택하거나 실수로 공유 에셋을 덮어쓰는 경우가 많습니다.

3단계 구조:

* 레벨 1: 제품 영역(클라우드/개발 도구/보안/데이터 엔지니어링)

* 2단계: 강의 이름

* 레벨 3: 에셋 유형(비디오/PDF/SCORM/퀴즈)

각 제품 팀에는 레벨 1 폴더에만 액세스할 수 있습니다. 특정 강의에 대한 특정 퀴즈를 찾는다는 것은 수천 개의 파일을 검색하기 보다는 정확히 올바른 Level 3 폴더로 이동한다는 것을 의미합니다. 보안 팀이 SCORM 패키지를 업데이트하면 보안 > [과정 이름] > SCORM에 있으며 실수로 다른 팀의 지점에 착륙할 수 없다는 것을 알게 됩니다.

### 관리자 권한으로 콘텐츠 폴더 관리

Adobe Learning Manager의 관리자는 콘텐츠 폴더 계층 구조를 만들고 유지하며 특정 폴더에 액세스할 수 있는 사용자 정의 역할을 제어하고 폴더 이름 및 삭제를 관리합니다. 작성자는 폴더에 콘텐츠를 추가하고 계층 내에서 콘텐츠를 구성할 수 있지만, 관리자만 폴더를 만들거나 이름을 바꾸거나 삭제할 수 있습니다.

#### 콘텐츠 폴더 생성

>[!NOTE]
>
>동일한 상위 항목 아래의 동일한 레벨에 있는 두 폴더는 이름을 공유할 수 없습니다. 같은 이름이 다른 분기나 다른 수준에서 허용됩니다.

1. 관리자 권한으로 Adobe Learning Manager에 로그인합니다.
2. 왼쪽 탐색에서 **구성** > **설정**&#x200B;을 선택합니다.
3. **고급** 섹션에서 **콘텐츠 폴더**&#x200B;를 선택합니다.
4. 페이지의 오른쪽 상단에서 **추가**&#x200B;를 선택합니다. **새 폴더 추가** 대화 상자가 열립니다.
5. 폴더의 이름 및 설명(선택 사항)을 입력합니다.
6. **저장**&#x200B;을 선택합니다. 폴더가 만들어지고 폴더 목록에 나타납니다.


#### 하위 폴더 만들기

1. **콘텐츠 폴더** 페이지에서 상위 폴더를 찾습니다.
2. 폴더 이름 옆의 **하위 폴더 만들기** 옵션을 선택합니다.
3. 하위 폴더의 이름 및 설명(선택 사항)을 입력합니다.
4. **저장**&#x200B;을 선택합니다. 하위 폴더는 폴더 목록에서 상위 폴더 아래로 들여쓰기되어 나타납니다.

>[!NOTE]
>
>각 폴더에는 최대 25개의 직접 하위 폴더를 포함할 수 있습니다. 레벨 3이 최대 깊이입니다. Level 3 폴더에는 하위 폴더를 만들 수 없습니다.

#### 폴더 이름 바꾸기

1. **콘텐츠 폴더** 페이지에서 이름을 바꿀 폴더를 선택합니다. 폴더가 편집 모드로 열립니다.
2. 폴더 이름과 필요한 경우 설명을 업데이트합니다.
3. **저장**&#x200B;을 선택합니다. 폴더가 새 이름으로 저장됩니다.

#### 폴더 삭제

삭제하기 전에 다음 규칙에 주의하십시오.

* 모든 수준에서 빈 폴더를 삭제할 수 있습니다.
* 다른 폴더에 연결되지 않은 내용이 포함된 폴더는 삭제할 수 없습니다. 먼저 해당 콘텐츠를 다른 폴더로 이동합니다.
* 상위 폴더를 삭제하면 해당 하위 폴더가 모두 삭제됩니다. 상위 폴더를 선택하면 모든 하위 폴더가 자동으로 선택됩니다.

#### 상위 폴더 삭제

1. **콘텐츠 폴더** 페이지에서 삭제할 각 폴더 옆의 확인란을 선택합니다.
2. 페이지의 오른쪽 상단에서 **작업** > **폴더 삭제**&#x200B;를 선택합니다.
3. 메시지가 표시되면 삭제를 확인합니다. 상위 폴더 내의 모든 하위 폴더도 삭제됩니다.

#### 하위 폴더 삭제

1. **콘텐츠 폴더** 페이지에서 삭제할 하위 폴더 옆의 확인란을 선택합니다.
2. 페이지의 오른쪽 상단에서 **작업** > **폴더 삭제**&#x200B;를 선택합니다.
3. 메시지가 표시되면 삭제를 확인합니다. 하위 폴더가 삭제됩니다.

>[!CAUTION]
>
>폴더 삭제는 영구적입니다. 확인하기 전에 폴더 내의 모든 콘텐츠가 다른 위치로 이동되었는지 확인하십시오.


#### 사용자 정의 역할에 대한 폴더 액세스 구성

사용자 정의 역할을 특정 수준 1 폴더로 제한하면 해당 역할을 가진 사용자 정의 관리자와 작성자가 관련 내용만 볼 수 있습니다.

**수준 1 폴더 수준에서만** 액세스 권한이 설정됩니다. 수준 1 폴더에 대한 사용자 정의 역할 액세스 권한을 부여하면 해당 역할은 해당 폴더 내의 모든 수준 2 및 수준 3 하위 폴더에 대한 액세스 권한을 자동으로 얻습니다. 하위 폴더 레벨에서는 개별적으로 액세스 권한을 할당할 수 없습니다.

1. 왼쪽 탐색에서 **사용자** > **사용자 지정 역할**&#x200B;을 선택합니다.
2. 구성할 사용자 정의 역할을 열거나 새 역할을 만듭니다.
3. **계정 권한**&#x200B;에서 **콘텐츠 폴더** 섹션을 찾습니다.
4. **선택한 폴더**&#x200B;를 선택합니다.
5. 이 역할이 액세스할 수 있어야 하는 수준 1 폴더를 선택합니다.
6. **확인**&#x200B;을 선택합니다.

이 역할을 가진 사용자는 선택한 수준 1 폴더와 그 하위 폴더의 콘텐츠만 볼 수 있습니다. 다른 비공개 폴더와 공개 폴더의 콘텐츠는 액세스할 수 없습니다.

#### 모범 사례

다음 방법을 통해 쉽게 확장할 수 있고 쉽게 탐색할 수 있는 폴더 구조를 만들 수 있습니다.

1. **폴더를 만들기 전에 구조를 계획합니다.** 콘텐츠가 계층화되면 많은 양의 콘텐츠를 이동해야 하는 구조조정이 이루어집니다. 시작하기 전에 제품 라인, 부서 또는 교육 프로그램과 같은 레벨 1 범주를 결정합니다.

2. **의미 있는 그룹화에는 세 가지 수준을 사용하십시오.** 일반적인 패턴은 다음과 같습니다. 광범위한 도메인 또는 프로그램의 경우 수준 1, 제공 유형 또는 팀의 경우 수준 2, 개별 에셋의 경우 수준 3 예:

   * 1단계: 영업 교육

   * 레벨 2: 자가 진행식 모듈

   * 레벨 3: PDF 에셋

3. **이름을 부모 안에서 짧고 설명적이며 고유한 이름으로 유지합니다.** &quot;Module 1&quot; 또는 &quot;Content&quot;와 같은 일반 이름은 사용하지 마세요. 라이브러리를 검색하는 작성자에게 적합한 식별자를 사용합니다.

4. **수준 1에서만 사용자 지정 역할 액세스 권한 할당** 액세스는 자동으로 캐스케이드되므로 레벨 1에서 할당하면 충분하며 액세스 관리가 간단합니다. 수준 2 또는 수준 3 하위 폴더를 추가할 때는 액세스 권한을 업데이트할 필요가 없습니다.

5. **폴더를 삭제하기 전에 콘텐츠를 이동합니다.** 폴더에 다른 곳에 연결되지 않은 콘텐츠가 있으면 삭제가 차단됩니다. 삭제하기 전에 폴더 내용을 검토하는 습관을 들입니다.
