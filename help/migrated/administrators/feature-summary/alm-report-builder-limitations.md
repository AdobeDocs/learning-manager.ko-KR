# Adobe Learning Manager Report Builder 제한 사항

Report Builder은 광범위한 사용자 정의 보고 요구 사항을 다루지만 이 릴리스에서는 일부 보고서 유형, 데이터 세트 및 구성이 지원되지 않습니다. 이 섹션에서는 알려진 제한 사항을 나열하고 가능한 경우 해결 방법을 제안합니다.

## 반복 인증

반복 인증과 관련된 특정 쿼리가 이 릴리스의 Report Builder에서 제대로 작동하지 않습니다. 여러 인증 기간에 걸쳐 재등록 또는 재완료를 추적하는 것과 같이 반복 인증 주기가 보고서에 포함된 경우 대신 Adobe Learning Manager의 기존 인증 보고서를 사용하십시오.

## 지원되지 않는 데이터 세트 및 보고서 유형

이 릴리스의 Report Builder에서 사용할 수 있는 항목은 다음과 같습니다.

* **비즈니스 결과 데이터**: Report Builder이 BYOD(Bring Your Own Data) 통합을 지원하지 않습니다. 외부 비즈니스 데이터와 LMS 데이터를 결합하는 보고서는 지원되지 않습니다.
* **감사 보고서**: 시스템 감사 추적 및 관리자 활동 로그를 Report Builder에 빌드할 수 없습니다. 이 데이터에 대해 관리자 보기에서 기존 감사 보고서를 사용합니다.
* **증분 보고서**: Report Builder은 매번 전체 스냅숏 보고서를 생성합니다. 마지막 다운로드 이후 변경된 레코드만 반환하는 증분 보고서는 지원되지 않습니다. 증분 데이터의 경우 작업 API를 사용합니다.

## 트렌드 보고서 제한 사항

### 추세 보고서당 하나의 날짜 열만

단일 추세 보고서는 하나의 날짜 기반 열로만 그룹화할 수 있습니다. 동일한 보고서에서 서로 다른 두 날짜 기반 추세를 결합할 수 없습니다.

예를 들어 다음과 같은 항목을 작성할 수 있습니다.

* **월별 등록**&#x200B;을 보여주는 보고서(등록 날짜별 그룹화)
* **월별 완료**&#x200B;를 표시하는 별도의 보고서(완료 날짜별 그룹화)

등록 월과 완료 월을 모두 별도의 추세 열로 표시하는 단일 보고서를 작성할 수 없습니다. 둘 다 가져오려면 두 개의 별도 보고서를 만들고 나란히 분석합니다.

### 추세는 기록 스냅샷이 아닌 현재 데이터를 반영합니다.

Report Builder의 추세 보고서는 기간별 스냅샷이 아닌 현재 시점 데이터를 기반으로 합니다. 보고서는 날짜 필드에 분산되어 있는 현재 기록 상태를 반영합니다.

이는 다음을 의미합니다.

* 1월에 등록했지만 나중에 등록 취소된 학습자는 1월 등록 추세에 나타나지 않을 수 있습니다
* 재구성한 사용자 그룹은 지난 몇 달 동안의 과거 멤버십을 반영하지 않습니다.
* 삭제된 레코드는 활성화된 기간의 추세에서 나타나지 않을 수 있습니다

방향 분석에 추세 보고서를 사용합니다. 정확한 기록 또는 감사 목적으로 기존의 고정된 보고서 또는 작업 API 내보내기를 사용합니다.
