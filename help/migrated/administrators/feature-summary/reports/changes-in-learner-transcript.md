# 4월 릴리스의 학습자 성적 증명서 변경 사항

## 완료 방법 열

**완료 방법** 열은 관리자의 학습자 성적 증명서에서 각 기록이 완료된 방식을 나타냅니다.

**값:**

1. **직접**(직접 완료용)
2. **대체**(대체 관계를 통해 달성된 완료용)
3. **대체 취소됨**(소급 미완료 및 관계 제거로 인해 모든 대체 완료가 취소된 경우)

>[!NOTE]
>
>이 열은 학습자의 LT에는 표시되지 않으며 보고 및 추적을 위한 책임자 LT에서만 사용할 수 있습니다.

**영향**: 강의 완료 방식과 관련하여 관리자가 명확한 감사 추적, 준수 추적 및 투명성을 사용할 수 있도록 합니다.

## 학습자 성적 증명서의 대체 완료 추적

대체 완료를 통해 학습자가 설정된 관계에 따라 동등한 소스 강의 또는 경로를 완료한 경우 대상 강의 또는 학습 경로에 대한 완료 점수를 받을 수 있습니다.

학습자 성적 증명서(LT)에서 대체 완료는 세 개의 기존 열(**상태**, **완료 날짜** 및 **완료 원본**)에 영향을 줍니다.

- **상태**: 대체 완료로 인해 학습자가 대상 과정 또는 경로를 직접 완료하지 않았더라도 상태가 **완료**&#x200B;될 수 있습니다. 기타 상태(**시작되지 않음**, **진행 중**, **등록 취소됨**)에는 영향을 주지 않습니다.
- **완료 날짜**: 대체 완료의 경우 해당 날짜는 원본 과정 또는 경로에서 상속됩니다. 나중에 학습자가 대상을 직접 완료하면 날짜가 업데이트되어 직접 완료한 날짜가 반영됩니다.
- **완료 원본**: 대체 완료를 제공한 원본 과정 또는 경로의 교육 ID를 캡처합니다. 여러 활성 소스가 쉼표로 구분된 ID로 나열됩니다. 소스가 취소된 경우 활성 소스만 유지됩니다. 소스가 여러 개인 경우 가장 빠른 완료 일자가 사용됩니다.

**영향**: 대체 완료를 통해 수동 조정을 줄이고, 학습 경로 및 인증의 진행 상황을 자동으로 추적하고, 규정 준수 요구 사항을 지원합니다.

>[!NOTE]
>
>학습자 성적 증명서에 **완료 방법** 열이 표시되지 않습니다. 이 열은 책임자 학습자 성적 증명서에서만 사용할 수 있습니다.

## 대체에 대한 완료 날짜 논리

학습자 성적 증명서의 **완료 날짜** 열은 직접 및 대체 완료에 모두 사용됩니다.

- 대체 완료의 경우 날짜는 학습자가 소스 과정 또는 경로를 완료한 날짜를 반영합니다.
- 나중에 학습자가 직접 대상을 완료하면 완료 날짜가 직접 완료 날짜로 업데이트됩니다.
- 대체 완료 날짜에 대해 새로 추가된 열이 없습니다.
- 복수 출처가 대체 완료를 제공하는 경우 가장 빠른 활성 완료 일자가 사용됩니다.
- 출처가 취소된 경우(소급 미완료 사용), 일자가 다음 최초 유효 출처로 갱신되거나 유효 출처가 남아 있지 않은 경우 지워집니다.

**영향**: 대체 관계가 시간이 지남에 따라 변경되는 경우에도 정확한 기록 추적 및 일관된 보고를 보장합니다.

## 해지된 대체 완료

취소되는 대체 완료는 소급 미완료가 활성화된 경우 대상 과정 또는 학습 경로에 대한 모든 소스 관계가 제거될 때 발생합니다.

### 트리거 조건

- 계정 레벨에서 소급 미완료를 사용으로 설정해야 합니다.
- **모든** 활성 원본 관계가 제거된 경우에만 취소가 발생합니다.
- 하나 이상의 소스가 남아 있으면 대체 완료가 지속되고 완료 소스 열이 그에 따라 업데이트됩니다.

### 학습자 성적 증명서에 미치는 영향

- **상태**: 모든 대체 완료가 취소되고 직접 완료가 없는 경우 상태가 업데이트됩니다(예: **완료됨**&#x200B;에서 **시작되지 않음** 또는 **진행 중**).
- **완료 날짜**: 활성 원본이 남아 있지 않고 직접 완료되지 않으면 지워집니다.
- **완료 원본**: 해지된 원본을 제거하도록 업데이트했습니다. 모든 원본이 해지되면 지워집니다.

학습자가 직접 완료한 경우 대체 소스를 취소해도 완료 상태 또는 완료 날짜에 영향을 주지 않습니다.

>[!NOTE]
>
>여러 소스가 대체 완료로 제공되었지만 일부만 취소된 경우 대본에는 나머지 활성 소스와 가장 빠른 완료 날짜가 반영됩니다. 모든 소스가 취소되고 직접 완료가 없는 경우 학습자는 대상에 대한 완료 상태를 상실합니다.

## 체크리스트 검토자 의견에 대한 보고 기능 향상

체크리스트 모듈의 검토자 의견이 이제 이름이 바뀐 열 **검토자 의견** 아래의 학습자 성적 증명서 보고서에 포함됩니다.

**영향**: 학습자와 관리자가 통합된 피드백을 볼 수 있도록 하여 투명도를 높이고 성능 평가를 지원합니다.

## 학습 시간 계산 개선

이제 학습자 성적 증명서 보고서에서 정제된 논리를 사용하여 사용자 활동과 브라우저 탭 포커스를 기준으로 활성 및 유휴 학습 시간을 구분합니다.

**영향**: 학습자 참여를 더 정확하게 측정하고 준수 보고 및 분석을 지원합니다.
