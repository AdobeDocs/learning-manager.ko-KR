# Report Builder에서 사용 가능한 데이터 집합

Report Builder은 사용 가능한 모든 열을 이름이 지정된 관련 필드 그룹인 데이터 세트로 구성합니다. 이 문서에서는 각 데이터 세트를 나열하고, 데이터 세트가 포함하는 내용을 설명하며, 단일 보고서에서 결합할 수 있는 데이터 세트에 대해 설명합니다.

**데이터 집합들이 함께 결합하는 방법**

모든 데이터 세트를 단일 보고서에 결합할 수 있는 것은 아닙니다. 데이터 세트는 조인할 수 있도록 Adobe Learning Manager의 데이터 모델에서 논리적 관계를 공유해야 합니다. 첫 번째 열을 추가하면 Report Builder이 나머지 데이터 세트를 필터링하여 호환되는 데이터 세트만 표시합니다. 데이터 세트가 회색으로 표시되면 이미 선택한 열에 직접 조인할 수 없습니다.

데이터 집합은 이미 선택한 열과 조인할 수 없습니다. 이것은 데이터 모델의 하드 제약 조건입니다. 두 데이터 집합은 호환되는 조인 경로를 공유하지 않습니다.

일반적인 예는 **준수 %** 파생 필드입니다. 준수 %를 선택하면 **성적 증명서**, **모듈 성적 증명서** 및 **LO 인스턴스** 데이터 집합이 비활성화됩니다. 준수 %는 학습 객체 및 카탈로그에 대해 사용자 또는 사용자 그룹 수준에서 계산됩니다. **사용자**, **사용자 그룹**, **학습 개체** 및 **카탈로그** 열 및 필터에만 사용됩니다.

비활성화된 데이터 세트를 사용하려면 충돌을 일으키는 열의 선택을 취소한 다음 필요한 데이터 세트를 추가합니다.

아래 조인 관계는 Report Builder 데이터 모델에서 그려집니다. 이러한 데이터 집합을 이해하면 보고서를 작성하기 전에 결합할 데이터 집합을 계획하는 데 도움이 됩니다.

**허브 데이터 집합**

두 개의 데이터 세트가 중앙 허브 역할을 합니다. 대부분의 다른 데이터 세트는 이러한 데이터 세트를 통해 연결됩니다.

* **등록**(등록 데이터 집합), 기본 팩트 테이블. **사용자**(등록한 학습자), **학습 개체**(등록한 항목)와 학습 개체를 통해 **모듈 세션**, **카탈로그**, **카탈로그 레이블** 및 **LO 인스턴스**&#x200B;에 직접 연결됩니다.
* **모듈 대본**(moduleTranscript 데이터 집합) - 모듈 수준 진행률 팩트 테이블입니다. **사용자** 및 **모듈 세션**&#x200B;에 연결되며, **학습 개체**&#x200B;에 연결됩니다.

학습자, 과정 및 완료 데이터를 결합하는 대부분의 보고서는 이러한 두 개의 허브 중 하나를 통해 빌드됩니다.

**사용 사례로 패스 연결**

**이러한 데이터 집합을 결합하려면**

**조인 경로**

**사용자** + **등록**

사용자 →(학습자 ID를 통해)

**등록** + **학습 개체**

학습 개체→ 등록(LO ID를 통해)

**학습 개체** + **카탈로그**

학습 개체 → LO 카탈로그 → 카탈로그

**학습 개체** + **카탈로그 레이블**

학습 객체 → LO 카탈로그 레이블

**학습 개체** + **LO 인스턴스**

학습 개체→ LO 인스턴스

**모듈 세션** + **학습 개체**

모듈 세션 → 학습 개체(세션 강의 이용)

**모듈 세션** + **사용자**(강사)

모듈 세션 → 사용자(강사 별칭 FK 사용)

**사용자** + **사용자 그룹**

사용자 그룹 멤버십 → 사용자 + 사용자 그룹

**학습 개체** + **사용자**(작성자)

LO Author → 사용자(작성자 별칭 FK 사용)

**사용자** + **활성 필드**

사용자 활성 필드 → 사용자(사용자 ID를 통해)

**별칭이 지정된 관계**

데이터 모델의 일부 필드가 직접 학습자 ID가 아닌 명명된 역할을 통해 **사용자** 엔터티에 가입합니다. 이것은 앨리어싱된 외래 키입니다. 이들은 동일한 사용자 테이블을 가리키지만 다른 역할을 나타냅니다.

* **강사**: 모듈 세션이 세션 강사로 사용자에게 참가합니다.
* **작성자**: LO 작성자가 학습 개체의 작성자로 사용자에게 참가합니다.
* **관리자**: 사용자가 직접 참여하여 학습자의 관리자를 나타냅니다
* **작성자**: 등록 및 모듈 성적 증명서에는 각각 별도의 &quot;작성자&quot; 사용자 참조가 포함됩니다.

단일 보고서가 학습자의 이름과 강사의 이름을 모두 표시할 수 있는 이유다. 둘 다 다른 연결 경로를 통해 사용자 엔티티에서 가져옵니다.

**사용자 그룹 데이터 집합 형식**

**사용자 그룹** 범주에는 각각 다른 유형의 그룹을 포함하는 몇 가지 개별 데이터 집합 보기가 포함되어 있습니다.

**데이터 집합**

**그룹 유형**

**사용자 그룹**(userGroup)

모든 사용자 그룹. 기본 사용자 그룹 데이터 집합으로 사용

**활성 필드 사용자 그룹**(active_field_user_group)

활성 필드 값(예: 지역, 부서)을 기준으로 그룹화

**팀 사용자 그룹**(team_user_group)

관리자 계층 기반 그룹

**사용자 지정 사용자 그룹**(custom_user_group)

수동으로 구성된 사용자 정의 그룹

**인빌드 사용자 그룹**(inbuilt_user_group)

시스템 정의 그룹

뷰 기반 사용자 그룹 데이터 집합(**활성 필드**, **팀**, **사용자 지정**, **인빌드**)은 스키마에 직접적인 외래 키 관계가 없으며 SQL 보기입니다. 즉, 핵심 **사용자 그룹** 데이터 집합보다 조인 제약 조건이 더 느슨합니다. 가장 신뢰할 수 있는 결과를 얻으려면 등록 또는 성적 증명서 데이터와 함께 사용자 그룹 데이터에 가입할 때 핵심 **사용자 그룹** 데이터 집합을 사용하십시오.
