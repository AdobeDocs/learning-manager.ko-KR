# 이 릴리스의 새로운 기능 {#whatsnewandchanged}

Adobe Learning Manager 2024년 7월 릴리스의 새로운 기능과 개선 사항에 대해 알아봅니다.

## 규정 준수 대시보드의 개선 사항

### 규정 준수 대시보드란? {#whatiscompliancedashboard}

관리자는 Adobe Learning Manager **의****[!UICONTROL 규정 준수 대시보드]**&#x200B;를 통해 학습자의 학습 목표 달성 상황을 모니터링하고 모니터링할 수 있습니다. 팀 구성원이 마감일을 준수하고 학습 프로세스를 따라가고 있는지 확인할 수 있으므로 규정 준수를 보장하는 데 도움이 됩니다. 관리자는 규정 준수 대시보드를 설정하고 관리자와 공유할 수 있습니다.

Admin 앱에서 규정 준수 대시보드에 액세스하려면 [보고서] > **[!UICONTROL [학습 요약]**] > **[!UICONTROL [규정 준수 대시보드]**]를 선택합니다&#x200B;**[!UICONTROL .]**

### 릴리스에서 변경되는 내용

관리자와 관리자는 향상된 규정 준수 대시보드를 통해 특정 범주(예: 영업, 마케팅, 법무)와 관련된 규정 준수 유형 과정, 학습 경로 또는 인증을 볼 수 있습니다. 관리자는 사용자 지정 규정 준수 과정을 특정 범주로 분류할 수 있습니다. 사용자 지정 규정 준수 범주는 카탈로그 레이블에 의해 제공됩니다.  관리자는 코스 대시보드를 만들고 관리자와 공유할 수 있습니다. 그러면 관리자는 해당 인스턴스에서 동일한 대시보드를 볼 수 있습니다. 또한 규정 준수 대시보드 및 규정 준수 전자 메일 알림의 사용자 인터페이스가 향상되었습니다.![](assets/compliance-dashboard-admin.png)

#### 워크플로

향상된 규정 준수 대시보드를 사용하는 단계는 다음과 같습니다.

| 책임자 | 사용자 지정 규정 준수 레이블 만들기 | 자세한 내용은 Create [custom compliance labels](/help/migrated/administrators/feature-summary/reports.md#compliance-dashboard) 문서를 참조하십시오 |
| 작성자 | 코스에 이러한 레이블을 추가합니다. | 자세한 내용은 이 문서 [(과정/학습 경로/인증](/help/migrated/authors/feature-summary/courses.md#add-compliance-labels-to-courselearning-pathcertification) 에 규정 준수 레이블 추가)를 참조하세요. |
| 책임자 | 규정 준수 과정으로 대시보드를 만들고 관리자와 공유 | 자세한 내용은 규정 준수 대시보드](/help/migrated/administrators/feature-summary/reports.md#create-and-share-a-compliance-dashboard) 만들기 및 공유 문서를 [참조하세요. |
| 관리자 | 규정 준수 대시보드 보기 | 자세한 내용은 규정 준수 상태](/help/migrated/managers/feature-summary/manager-dashboard.md#compliance-status) 문서를 [참조하십시오 |

## 로그인하지 않은 환경에 대한 업데이트

로그인하지 않은 환경을 사용하면 로그인하지 않은 고객을 위한 실시간 환경을 만들 수 있습니다. 이것은 마케팅 캠페인의 랜딩 페이지 역할을 하며 가입을 장려하기에 충분한 정보를 제공합니다.

### 이 릴리스에서 변경되는 내용

고객은 프리미엄 플랜을 구매하여 확장성이 뛰어난 로그인하지 않은 환경을 구축할 수 있습니다. Training Data Access](/help/migrated/integration-admin/feature-summary/connectors.md#training-data-access)를 [기반으로 하는 이 로그되지 않은 경험은 Adobe Learning Manager API를 사용하여 좌석 제한, 점유된 좌석, 대기자 명단 제한 및 대기자 명단 수에 대한 실시간 데이터를 제공합니다. 고객은 이러한 API를 사용하여 로그인하지 않은 학습자에게 검색 및 필터링 기능과 전체 과정 요약을 제공할 수 있습니다.

>[!NOTE]
>
>지원 팀 또는 CSAM 문의하여 프리미엄 플랜을 구매하시기 바랍니다.

## 이 릴리스의 API 변경 사항

### 검색 API

검색 API에는 다음과 같은 변경 내용이 포함되어 있습니다.

학습자는 API를 사용하여 카탈로그 필터 내에서 태그를 검색할 수 있습니다 ```GET /search``` . 학습자는 매개 변수 값으로 ```filter.loTypes``` 선택하여 ```tag``` 태그를 검색할 수 있습니다.

**샘플 curl**

```
curl -X GET --header 'Accept: application/vnd.api+json' --header 'Authorization: oauth 5a858f23924f4feafa38ae8d6c4d97b6' 'https://example.com/primeapi/v2/search?page[limit]=10&query=Business&autoCompleteMode=true&filter.loTypes=tag&sort=relevance&filter.ignoreEnhancedLP=true&matchType=phrase&persistSearchHistory=true&stemmed=false&highlightResults=true'
```

새 필터, 사용 가능한 좌석, 사용 가능한 대기자 명단 및 시간 범위 필터가 다음 API ```GET /search``` `GET /learningObjects`에 추가되었습니다.

새 필터가 `filter.session.includeEnrollmentDeadline` 다음 API ```GET /search```에 추가되었습니다 .

### 계정 API

새로운 열`custom_injections`, , 및 `complianceLabelDefaultID` 가 API에 ```GET /account``` 추가되어 사용자 끝점의 `showComplianceLabel`계정 데이터를 가져옵니다.

### 학습 객체 API

다음은 이 업데이트에서 학습 객체 API에 적용된 변경 내용입니다.

새 응답, 레거시 작성자 ID 및 API에 추가된 `authorDetails` 기타 세부 정보입니다 `GET /learningObjects` . 또한 기존 작성자와 해당 과정을 필터링하기 위해 새 필터 `filter.authors`이(가) 추가되었습니다.

라는 `effectivenessIndex` 새 속성은 코스 효과 데이터를 가져오는 데 도움이 됩니다.

**샘플 curl**

```
curl --location 'https://example.com/primeapi/v2/learningObjects/course%3A9790045?enforcedFields%5BlearningObject%5D=effectivenessData' \
--header 'Accept: application/vnd.api+json' \
--header 'Authorization: oauth 598665ab5c8a99bea0e774d9faf7f3ca'
```

이 과정을 수강해야 하는 사용자에 대한 세부 정보를 제공하는 새 응답 `whoShouldTake`이 `POST /learningObjects/query`, `GET /learningObjects/{id}`, 및 `GET /learningObjects`.

**샘플 curl**

```
curl -X GET --header 'Accept: application/vnd.api+json' --header 'Authorization: oauth 28a83fb8c87579af8ebc4434cc80f0c0' 'https://example.com/primeapi/v2/learningObjects/course%3A1131255' 
```

대기자 명단 제한에 대한 세부 정보를 제공하는 새 응답 `waitlistLimit`이 API에 `GET /learningObjects` 추가되었습니다.

학습 객체의 총 개수를 제공하는 새로운 응답 `count` 이 API `GET/ learningObjects` 및 `POST/ learningObjects/query`.

새 응답 `catalogFieldId` `fieldValueId` 이 API에 `GET/ learningObjects` 추가되었습니다`catalogLabels`.

학습자는 API `GET /preview/learningObjects`에서 카탈로그 레이블 값을 가져올 수 있습니다.

### 마켓플레이스 수를 가져오는 새로운 API

이 릴리스에서는 새 API `GET /search/marketplace/count` 가 추가되었습니다. 이렇게 하면 Content Marketplace에서 사용 가능한 학습 객체의 수를 파악하는 데 도움이 됩니다.

**샘플 curl**

```
curl -X GET --header 'Accept: application/vnd.api+json' --header 'Authorization: oauth d8631c7b0e3b5d2ae00422ef30aaecfc' 'https://example.com/primeapi/v2/search/marketplace/count?query=course'
```

**샘플 응답**

```
{
  "count": 54910
}
```

### 학습 개체 인스턴스 API

다음은 이 업데이트에서 학습 객체 인스턴스 API에 적용된 변경 사항입니다.

이 릴리스에서는 라는 `gamificationEnabled` 새 키가 학습 객체 인스턴스 API `GET /learningObjects/{loId}/instances/{loInstanceId}`에 추가되었습니다.

**샘플 curl**

```
curl --location 'http://example.com/acapapi/primeapi/v2/learningObjects/learningProgram:12756/instances/learningProgram:12756_15644' 
```

게이미피케이션 설정의 세부 정보를 가져오기 위한 위 API의 새 `gamificationSettings` 속성입니다. 예: `GET /learningObjects/{loId}/instances/{loInstanceId}/gamificationSettings`.

**샘플 curl**

```
curl --location 'http://example.com/acapapi/primeapi/v2/learningObjects/learningProgram:103852/instances/learningProgram:103852_103526/gamificationSettings'
```

게이미피케이션 설정의 세부 정보를 가져오기 위한 위 API의 새 `leaderboard` 속성입니다. 예: `GET /learningObjects/{loId}/instances/{loInstanceId}/leaderboard`.

**샘플 curl**

```
curl --location 'https://example.com/primeapi/v2/learningObjects/learningProgram:106339/instances/learningProgram:106339_105775/leaderboard' \
--header 'Accept: application/vnd.api+json' \
--header 'Authorization: oauth de4b5ee6efdd42375130db27ff503dd4'
```

### 오프셋 제한 변경

시스템 성능을 개선하고 리소스 사용률을 보다 효과적으로 관리하기 위해 Adobe에서는 ADMIN 및 LEARNER 범위 모두에 대해 GET /users 끝점에서 더 이상 높은 오프셋 값을 사용하지 않습니다. Jobs API를 사용하여 오프셋 값이 있는 레코드를 검색하는 것이 좋습니다.

### 더 이상 사용되지 않는 API

제품에서 사용되지 않는 모든 API의 누적 목록에 대한 Adobe Learning Manager](/help/migrated/api-deprecations-list.md)의 API 사용 중단을 확인하십시오[.

* 진행률(%)

Admin > Reports > Custom Reports **의 교육 보고서**&#x200B;와 작업 API **에는** Skill(s) 및 **Tag(s)****라는**&#x200B;열이 있었습니다. **** **** 이러한 열의 이름은 이제 기술&#x200B;**및**&#x200B;태그&#x200B;**로**&#x200B;변경되었습니다.

### 콘텐츠 감사 보고서

이 릴리스에서는 **[!UICONTROL 컨텐츠 감사 내역]** 보고서의 수정 유형 열에 다음과 같은 새로운 속성이 포함됩니다.

* 사용자 그룹 추가
* 사용자 그룹 제거
* 사용자 지정 레이블 추가
* 사용자 정의 레이블 제거
* 공유 카탈로그 추가
* 공유 카탈로그 제거
* 공유 카탈로그 업데이트

## 이 업데이트에서 수정된 버그

**활동 제출**

* 활동 제출 모듈에 파일을 다시 업로드하려고 하면 네트워크 호출에서 오류 500이 발생하면서 실패합니다.

**API**

* 여러 강사가 동일한 이메일 주소를 가지고 있는 경우 Connect VC 미팅을 만드는 데 실패합니다.
* 학습 경로에 등록한 후 MS Teams VC는 개요 페이지에 잘못된 URL을 표시합니다.
* Job API 응답의 일부로 제공된 사용자 보고서 미리 서명된 URL은 6시간 후에 만료됩니다.
* 코스에 대한 등록 보고서를 생성하는 동안 코스 이름 열에 잘못된 코스 이름이 표시됩니다.
* 마이그레이션 작업자는 코스에 대한 대량 API를 호출할 때 고유한 lo id를 보내지 못하지만 ID는 제거됩니다.
* 사용자가 액세스할 수 있는 특정 카탈로그에 강의가 포함되어 있는 경우(기본 카탈로그는 비활성화되어 있음) 등록되지 않은 학습자가 강의를 볼 수 없도록 설정되었음에도 불구하고 learningobject/id 끝점을 통해 강의의 메타데이터를 검색할 수 있습니다.
* GET /learningObject API에서 skillname의 이름에 쉼표가 있는 경우 skills 필터가 예상대로 작동하지 않습니다.
* SFTP에 대한 데이터 보존 작업자에 있는 파일의 타임스탬프 메타데이터에 불일치가 있습니다.
* 커넥터를 제거하고 다시 구성하면 프로젝트 마이그레이션 상태가 닫힌 것으로 나타납니다.
* 교육 보고서에는 &quot;태그&quot; 대신 열 머리글로 &quot;태그&quot;가 있습니다.
* 카탈로그가 비활성화되어 있고 내보낸 강좌 중 하나라도 비활성화된 카탈로그의 일부일 경우 Commerce 커넥터 내보내기가 실패합니다.

**인증**

* 반복 인증에 사용자를 다시 등록하는 데 실패하는 경우가 있습니다.

**사용자 지정 역할**

* 경우에 따라 사용자 정의 관리자가 강사 역할로 전환하려고 하면 오류 403 사용할 수 없음이 표시됩니다.

**전자 메일 템플릿 및 알림**

* 세션이 취소된 후 이메일 알림은 강사가 세션에서 제거될 때 마지막 강사 집합에게 전송되지 않습니다.
* 이끌이는 가상 강사 진행 교육을 만든 후 MS Teams에 대한 전자 메일 알림을 받지 않습니다. 과정이 게시되고 전자 메일 템플릿이 활성화된 후에만 전자 메일이 트리거됩니다.
* 경우에 따라 이메일 템플릿이 잘못된 날짜 형식과 번역으로 구성되어 있습니다.

**학습자**

* 학습자가 강의의 여러 인스턴스에 등록되어 있는 경우 출석 보고서를 다운로드하면 보고서에 잘못된 정보가 포함됩니다.
* 공개 스토리에 추가된 다른 사용자의 비공개 게시물을 볼 수 있습니다.
* 인증에서 학습자를 등록 취소할 수 없는 경우도 있습니다. 등록을 취소하려고 할 때 오류 메시지가 표시됩니다.
* 인증은 관리자가 하나의 과정만 선택한 후 완료로 표시한 후에도 완료로 표시됩니다.
* 관리자는 세션의 종료 시간이 이전 날짜로 변경된 경우 VC를 완료로 표시할 수 없습니다.
* 세션 출석 보고서는 대기자 명단에 있는 학습자의 경우 &quot;참석하지 않음&quot;으로 표시됩니다.

**학습자 앱**

* 강좌 노트를 PDF로 다운로드하면 노트가 무작위로 나타납니다. 그들은 명령을 따르지 않습니다.

**학습 경로**

* 학습 경로에서 기술을 선택한 후 텍스트 필드를 선택할 때 드롭다운이 예상대로 표시되지 않습니다.
* 경우에 따라 학습 경로에서 기술을 제거할 수 없습니다.

**학습 프로그램**

* 유연한 학습 프로그램에 많은 강의가 있는 경우 관리자가 완료로 표시한 후에도 학습 계획이 완료되지 않습니다.
* 등록 보고서의 last_modified_by 열은 학습자가 인스턴스를 변경할 때 업데이트되지 않습니다.

**신고**

* 경우에 따라 관리자가 교육 보고서를 내보낼 수 없습니다.
* SCORM 콘텐츠에 32,767자를 초과하는 질문이나 대답이 포함되어 있으면 Excel에서 과정 퀴즈 보고서를 다운로드할 수 없습니다.
* 게임화 재설정을 선택한 후에는 레벨 달성 날짜가 재설정되지 않습니다.

**검색**

* 현재는 모든 사용자 그룹을 내보낸 후 삭제된 사용자 그룹도 출력에 포함됩니다.
* 간헐적인 검색 문제로 인해 인증을 검색할 수 없습니다.
