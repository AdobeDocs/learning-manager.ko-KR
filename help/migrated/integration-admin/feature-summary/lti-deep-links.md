# Adobe Learning Manager의 LTI 딥 링크

## 개요

**다음 섹션은 관리자용입니다**

LTI 딥 링크는 강사 또는 강의 작성자가 Adobe Learning Manager(ALM)의 특정 학습 항목을 외부 LTI 도구 소비자/플랫폼(예: Canvas 또는 Moodle) 강의로 직접 검색, 선택 및 포함할 수 있는 LTI 이점 기능입니다.

LTI 딥 링크는 무들과 같은 학습 플랫폼에 과정을 추가하는 과정을 단순화한다. 현재 작업 과정에서 작성자는 UUID 내보내기 매개 변수를 포함하여 과정 URL을 수동으로 복사한 다음 과정 링크를 구성하는 동안 필요한 세부 정보를 LMS에 붙여넣어야 합니다. 이 단계는 모든 강의와 모든 배치에 대해 반복되어야 합니다. 예를 들어 10개의 다른 위치에 동일한 강의를 추가해야 하는 경우 작성자는 복사 및 붙여넣기 프로세스를 10번 반복해야 합니다. 이러한 수작업 방식은 수작업을 증가시키고 구성 오류의 위험을 높입니다.

딥 링크는 설정 과정에서 LMS가 과정 선택을 처리할 수 있도록 허용하여 이 오버헤드를 제거하고, 콘텐츠 선택에 적합한 실행 URL을 제공합니다.

이 모델에서:

* 외부 LMS의 강사와 작성자는 전용 딥 링크 선택 경험을 실행하여 ALM을 검색할 수 있습니다.
* 시스템은 선택한 항목이 강의 작성 작업 과정의 일부로 포함될 수 있도록 ALM에서 외부 LMS로 딥 링크 개체를 반환합니다.
* 학생들은 기본 LMS에서 심층 링크된 콘텐츠를 수료하며, ALM에서 호스팅되는 자료를 원활하게 실행합니다.

## 문제 진술

ALM은 현재 LTI 1.3 통합을 지원하지만 완전한 딥 링크 워크플로우가 없으면 강사와 작성자는 다음을 수행할 수 있는 구조화된 방법이 없습니다.

* 모달에서 전용 딥 링크 선택 환경을 실행합니다.
* 특정 플랫폼에 대해 표시되어야 하는 학습 개체만 찾아봅니다.
* 플랫폼에서 특정 학습 객체를 선택합니다.
* ALM은 강의에 직접 포함할 수 있도록 해당 학습 개체를 플랫폼으로 반환합니다.

이 기능이 없는 경우:

* 콘텐츠 선택이 수동 또는 조각화됨
* 모든 계정 내용은 명시적으로 필터링하지 않는 한 의도치 않게 노출될 수 있습니다
* 도구 공급자 통합은 운영하기가 더 어렵습니다
* 강의 작성자는 일관된 관리 워크플로를 통해 외부 LTI 콘텐츠를 포함할 수 없습니다

## 목표

이 기능의 주요 목적은 다음과 같습니다.

1. LTI 도구 공급자에서 LTI 딥 링크 활성화
   * ALM에서 LTI 도구 공급자로의 딥 링크 실행을 지원합니다.
2. 관리되는 콘텐츠 선택 워크플로우 제공
   * 딥 링크 선택 시 승인되고 관련 있는 카탈로그와 콘텐츠만 표시합니다.
3. 강사 및 작성자가 학습 개체를 선택할 수 있도록 허용
   * 적합한 학습 개체를 선택할 수 있도록 검색 및 필터링 가능한 UI를 제공합니다.
4. ALM에 대한 유효한 딥 링크 응답 반환
   * deep_link_return_url 매개 변수와 필요한 deep-link 페이로드를 사용하여 사용자를 플랫폼으로 다시 리디렉션합니다.
5. 플랫폼별 카탈로그 노출 지원
   * 관리자가 어떤 카탈로그가 어떤 LTI 플랫폼에 노출되는지 제어할 수 있도록 허용합니다.

## 담당자 및 역할

LTI 딥 링크 워크플로우에는 다음과 같은 사인이 포함됩니다.

| 담당자 | 설명 |
|---|---|
| 강사 또는 작성자 | 강의를 생성 또는 관리하고 딥 링크 선택 흐름을 실행하여 외부 콘텐츠를 포함합니다. |
| 통합 책임자 | LTI 도구를 등록 및 관리하고 딥 링크 비헤이비어를 활성화 및 구성합니다. |
| Prime에서 | 딥 링크 워크플로우를 통해 추가된 콘텐츠를 실행하고 소비합니다. |

*각 가상 사용자는 구성에서 소비에 이르기까지 딥 링크 워크플로우의 개별 단계에 매핑됩니다.*

## 데이터 및 매개 변수 요구 사항

딥 링크는 ALM과 LTI 플랫폼 간에 다음 매개변수를 교환합니다.

| 매개 변수 | 용도 |
|---|---|
| `deep_link_return_url` | 선택한 딥 링크 개체를 ALM으로 다시 보내는 데 사용되는 반환 끝점 |
| `accepted_types` | 플랫폼에서 허용하는 리소스 유형을 정의합니다. |
| `accept_multiple` | 여러 리소스 선택이 허용되는지 여부를 나타냅니다. 도구별로 구성할 수 있습니다. |
| `auto_create` | 플랫폼에서 연결된 리소스 항목을 자동으로 만들 수 있음을 나타냅니다. |

*이러한 매개 변수는 표시되는 내용과 선택 항목을 ALM으로 반환하는 방법을 제어합니다.*

## 딥 링크 만들기

### 선행 작업

1. 통합 책임자로 로그인해야 합니다.
2. LTI 통합을 설정하는 동안 딥 링크 지원 확인란을 선택합니다.
3. 사용자 또는 작성자를 선택 영역으로 이끌 필드에 URL을 입력합니다.
4. 변경 내용 저장을 선택합니다.

   동일한 실행 URL을 재사용하여 구성 및 사용을 단순화합니다.

   동작은 LTI 메시지 유형에 따라 결정됩니다. 메시지 유형이 `content_consumption`이면 사용자는 강의 플레이어로 이동합니다. 메시지 유형이 `content_selection`인 경우 사용자는 딥 링크 플로우를 통해 라우팅됩니다. 여기서 작성자는 강의별 식별자를 수동으로 복사하지 않고 원하는 콘텐츠를 직접 선택할 수 있습니다.

   변경 내용을 저장한 후 **콘텐츠 선택** 탭을 선택합니다. 이 확인란을 선택한 후에만 **콘텐츠 선택** 탭이 활성화됩니다.

**다음 섹션은 작성자용입니다.**

작성자는 **콘텐츠 선택** 창에서 콘텐츠를 선택할 수 있습니다. **콘텐츠 선택** 창에 **카탈로그**, **강의 수** 및 **내보내기 날짜**&#x200B;가 표시됩니다.

1. 외부 통합 도구로 이동합니다.

   ![](assets/image_0005.png)

2. **카탈로그**&#x200B;를 선택하고 각 강의 옆의 확인란을 선택하여 딥 링크할 강의를 선택합니다. 여러 강의를 추가하는 경우 확인할 수 있는 확인 팝업이 나타납니다.

   ![](assets/image_0002.png)

   ![](assets/image_0003.png)

3. **콘텐츠 추가**&#x200B;를 선택합니다. **콘텐츠 추가**&#x200B;를 선택하면 모든 필드가 채워집니다. 사용자 정의 매개 변수 필드에서 내보내기 UUID를 볼 수 있습니다. 이전 단계에서 여러 강의를 선택한 경우 확인 메시지가 표시됩니다.

   ![](assets/image_0004.png)

4. 이때 다른 강의를 선택하거나 변경하고 싶은 경우 **취소**&#x200B;를 선택하고 **콘텐츠 선택** 탭으로 돌아가거나, **저장 후 반환**&#x200B;을 선택하거나 **저장 및 표시**&#x200B;를 선택할 수 있습니다. 딥 링크가 대상에 추가됩니다.

   ![](assets/image_0001.png)
