# Adobe Learning Manager 2026년 8월 릴리스의 변경 사항 보고

Adobe Learning Manager의 2026년 8월 릴리스에는 적응형 과정, 학습자 성적 증명서, 외부 학습, Gen AI 크레딧 사용 등에 대한 보고 기능이 개선되었습니다. 이 문서에서는 이 릴리스에서 관리자가 사용할 수 있는 새로운 열, 보고서 및 동작 변경 사항을 요약합니다.

## 변경된 내용

보고 업데이트는 적응형 과정 동작, 적응형 대기자 명단, 그레이드북 점수, 외부 학습, 증분 사용자 내보내기, Gen AI 크레딧 사용, 루트 인증 추적 및 Webhook 타임스탬프 정렬의 8개 기능 영역에 걸쳐 있습니다. 변경 사항은 다음 보고서에 가장 큰 영향을 미칩니다.

- 학습자 성적 증명서(LT)
- 교육 보고서
- 등록 보고서
- 대기자 명단 보고서
- 콘텐츠 감사 보고서

대부분의 업데이트에는 새 열이 추가됩니다. 일부 새로운 보고서 유형이 도입되었습니다. 일부는 기존 데이터를 모델링하거나 서식을 지정하는 방법을 변경했습니다.

## 적응형 과정 보고 변경 사항

### 교육 보고서

교육 보고서의 세 개의 새 열이 적응형 강의 동작을 지원합니다.

| **열** | **설명** | **지원되는 값** |
|--------------------------|----------------------------------------------------------|------------------------------------------------------------------------|
| 적응형 학습 목표 | 강의가 적합한지 여부를 식별합니다. | 참(적응형), 거짓(비적응형) |
| 가시성 사용자 그룹 | 각 모듈을 볼 수 있는 사용자 그룹을 나열합니다. | 하나 이상의 사용자 그룹 이름(예: All Learners, UG-Australia) |
| 필수 | 사용자 그룹의 필수 모듈인지 여부를 나타냅니다. | 모듈이 필수인 사용자 그룹 이름입니다. 비어 있음 = 선택 사항입니다. |

**가시성 사용자 그룹**&#x200B;과 **필수**&#x200B;을 결합하여 보고서에서 적응 완료 규칙을 직접 해석할 수 있습니다. 예를 들어 **모든 학습자**&#x200B;에게 모듈이 표시될 수 있지만 **관리자 그룹**&#x200B;에 대해서만 필수 모듈입니다.

### 학습자 성적 증명서

새 **이전 완료** 열은 적응형 로직이 다시 완료를 트리거할 때 기록 완료 데이터를 캡처합니다.

| **하위 필드** | **설명** |
|-----------------------|-----------------------------------------|
| completionRefreshDate | 완료가 재설정된 때의 타임스탬프 |
| completedDate | 이전 완료 타임스탬프 |
| progressAtFresh | 재설정 전 학습자 진행률 |
| gradeAtRefresh | 재설정 시 학습자 점수 |

이제 학습자 성적 증명서는 여러 완료 주기를 지원합니다. 강의 업데이트나 새로운 필수 모듈 등으로 인해 재완료 이벤트가 발생하면 이전 완료는 **이전 완료** 열로 이동합니다. 현재 완료는 표준 성적 증명서 필드에 남아 있습니다.

### 등록 보고서

새로운 **대기자 명단** 열은 학습자가 과정 내의 모든 모듈에 대기자 명단 상태인지 여부를 나타냅니다.

| **값** | **의미** |
|-----------|---------------------------------------------------------|
| 참 | 학습자가 하나 이상의 모듈에 대기자 명단에 있습니다 |
| false | 학습자가 보이는 모든 모듈에 등록을 확인했습니다 |

### 대기자 명단 보고서

두 개의 새 열과 향상된 상태 세부 지원 모듈을 통해 모듈 수준에서 대기자 명단 추적을 사용할 수 있습니다.

| **열** | **설명** |
|-----------------|----------------------------------------------------------------------------------------------------------------------------------------|
| **모듈** | 학습자가 대기자 명단에 있는 모듈(강의실 또는 가상 강의실 세션)의 이름 [인스턴스 상태] 열 뒤에 나타납니다. |
| **모듈 ID** | 학습자가 대기자 명단에 있는 모듈의 식별자입니다. 모듈 열 뒤에 나타납니다. |
| **포함 대상** | 이 강의가 포함된 학습 경로의 이름 및 ID 강의가 학습 경로에 포함되어 있지 않으면 비어 있습니다. |

대기자 명단 보고서가 강의 레벨 모델에서 모듈 세션 레벨 모델로 전환되었습니다. 이제 학습자를 일부 모듈에 등록하고 다른 모듈에 대기자 명단에 등록할 수 있습니다. 이 보고서는 또한 사용자 수 제한이 모듈 수준에서 적용되는 Flex 학습 경로 내의 대기자 명단 추적을 지원합니다.

### LP 등록 보고서

학습 경로 등록 보고서에는 새 **설명** 열도 포함됩니다. 학습자가 학습 경로를 구성하는 강의 내의 모든 강의실 또는 가상 강의실 세션에서 대기자 명단 상태에 있을 때 설명 열에는 **대기자 명단**&#x200B;이 표시됩니다. 모든 세션이 확정되면 해당 열은 비어 있습니다.

### 출석 보고서

이제 **학습자 상태** 열은 확인된 학습자와 대기자 명단에 등록된 학습자를 구분합니다.

| **값** | **의미** |
|------------|----------------------------------------|
| 확인됨 | 학습자에게 할당된 시트가 있습니다 |
| 대기자 명단 등록됨 | 학습자가 시트 할당 보류 중입니다. |

## 그레이드북 보고 변경 사항

### 학습자 성적 증명서

새로운 **가중치** 열은 전체 강의 점수에 대한 점수를 매길 수 있는 각 모듈의 기여도를 나타냅니다.

| **값** | **설명** |
|----------------------------------------------|------------------------------------------------------|
| 숫자 백분율(예: 20, 30, 50) | 강의 점수에 대한 모듈의 기여도 |
| 비어 있음 | 모듈이 점수를 매길 수 없음(예: PDF 또는 비디오) |

### 콘텐츠 감사 보고서

두 개의 새로운 이벤트는 그레이드북 구성 변경 사항을 캡처합니다.

| **이벤트** | **다음 경우에 트리거됨** | **캡처된 데이터** |
|-----------------------|-----------------------------------------------------------------|----------------------------------------------------------|
| Gradebook 업데이트됨 | 강의 레벨에서 그레이드북이 활성화, 비활성화 또는 수정됨 | 그레이드북 상태 변경, 그레이딩 구성 업데이트 |
| 모듈 두께 업데이트됨 | 모듈에 할당된 가중치가 수정됨 | 모듈 식별자, 업데이트된 가중치 값 |

학습자 성적 증명서에는 최신 가중치가 반영됩니다. 컨텐츠 감사 보고서는 기록 변경 사항을 추적합니다. 함께, 그들은 현재의 득점 논리와 그것이 어떻게 진화해 왔는지에 대한 완전한 그림을 당신에게 줍니다.

## 외부 학습 보고 변경 사항

### 학습자 성적 증명서

외부 학습 기록을 지원하기 위해 세 개의 새로운 열이 추가되었습니다.

| **열** | **설명** |
|------------------------|-----------------------------------------------------------------------------------------------------|
| 외부 학습명 | 학습자가 제출한 외부 학습 활동의 이름 |
| 사용자 정의 필드 | 외부 학습에 대해 구성된 사용자 정의 필드당 하나의 열(텍스트, 숫자, 확인란 또는 드롭다운) |
| 완료 주석 | 승인 또는 거부 중 입력한 관리자 설명 |

**참고:** 학습자 성적 증명서(학습자 셀프 서비스 보기)에서 열 배치가 관리자 학습자 성적 증명서와 다릅니다.

- **외부 학습 이름**&#x200B;은(는) 기존 **모듈** 열 바로 뒤에 추가됩니다.

- **완료 설명**&#x200B;은 기존 **검토자의 발언** 열 바로 뒤에 추가됩니다.

- 사용자 정의 필드 열(구성된 사용자 정의 필드당 하나)은 성적 증명서의 끝에 추가됩니다.

책임자 학습자 성적 증명서에서 외부 학습 이름 및 완료 설명을 포함한 모든 새 열이 맨 뒤에 사용자 정의 필드 열이 추가됩니다.

### 학습자 성적 증명서에 열 입력

이제 외부 학습 항목이 책임자 LT의 기존 학습 객체(강의, 학습 경로, 인증)와 함께 나타납니다. **유형** 열에는 쉽게 필터링할 수 있도록 새로운 외부 학습 분류가 포함되어 있습니다.

외부 학습 데이터는 학습자 성적 증명서와 책임자 LT 양쪽으로 흐릅니다. 완료 날짜, 상태 및 점수와 같은 핵심 필드가 기존 열에 매핑됩니다. 사용자 정의 필드는 추가 열로 추가됩니다.

## 증분 사용자 보고서 변경 사항

새로운 증분 내보내기 모델을 사용하면 매번 전체 데이터 내보내기를 생성하지 않고 지정된 기간 내에 데이터가 변경된 사용자만 내보낼 수 있습니다.

| **내보내기 모드** | **동작** |
|--------------------|-----------------------------------------------------------------|
| 전체 내보내기 | 계정의 모든 사용자를 반환합니다. |
| 증분 내보내기 | 지정된 날짜 범위 내의 변경 내용이 있는 사용자만 반환합니다. |

증분 내보내기를 사용하려면 **시작 날짜** 및 **종료 날짜**&#x200B;를 기준으로 필터링하여 변경 창을 정의하세요. 이제 데이터 플랫폼 파이프라인을 사용하여 사용자 보고서가 생성되고 큰 계정을 지원하기 위해 출력이 청크 단위로 반환됩니다.

새로운 크레딧 대시보드와 두 개의 보고서를 통해 책임자는 일반 AI 크레딧 소비를 확인할 수 있습니다.

### 크레딧 대시보드

대시보드는 계정 레벨에서 다음 측정 단위를 표시합니다.

| **메트릭** | **설명** |
|-------------------|---------------------------------------------------|
| 구매한 크레딧 | 계정에 대해 프로비저닝된 총 크레딧 |
| 사용된 크레딧 | AI 기반 기능에서 소비된 크레딧 |
| 남은 크레딧 | 소비 후 사용 가능한 크레딧 |
| 기능별 사용 | 개별 AI 기능으로 분할된 신용 소비 |

### 신규 보고서

| **신고** | **설명** |
|----------------------|---------------------------------------------------------------------------------------------|

| 감사 내역 보고서 | 사용자 식별자, 기능 이름, 사용한 크레딧, 타임스탬프 등 사용자 수준의 세부 정보를 제공합니다. |

## 기타 동작 변경 사항

### 루트 인증: 루트 교육 ID

**관리자 학습자 성적 증명서** 및 **학습자 성적 증명서**(학습자 셀프 서비스 보기) 끝에 새 **루트 교육 ID** 열이 추가됩니다. 인증의 모든 반복을 단일 루트 엔터티에 연결하는 고유 식별자를 캡처합니다. 이를 통해 추적과 필터링을 위해 인증의 모든 반복 인스턴스를 단일 루트 ID와 연결할 수 있습니다.

### Webhook 및 학습자 성적 증명서 타임스탬프 표준화

이제 Webhook 타임스탬프가 학습자 성적 증명서 서식에 맞춰 조정됩니다. 이제 Webhook 페이로드의 **데이터 개체** 내의 모든 날짜 및 시간 필드의 초 값이 00으로 설정되어 학습자 성적 증명서 보고서와 일치하는 분 단위의 세분성을 제공합니다. 이렇게 하면 Webhook 데이터를 학습자 성적 증명서 데이터와 비교할 때 타임스탬프 형식을 표준화할 필요가 없습니다.

### 공유 강의에 대한 API 응답의 작성자 정보

카탈로그 공유를 통해 강의를 한 Adobe Learning Manager 계정에서 다른 계정으로 공유하면 API의 학습 개체(LO) 응답이 이제 소스 계정의 원본 작성자 세부 정보만 반환합니다. 이전에는 수락하는 계정의 관리자가 해당 계정의 API 응답에 강의 작성자로 나타났습니다

이 변경 사항은 피어(수신) 계정에만 영향을 줍니다. 수신 계정에서 공유 강의의 LO 세부 정보 끝점을 쿼리하면 이제 authorNames 필드에 수신 계정의 관리자가 아닌 소스 계정의 원래 작성자가 반영됩니다.

소스 계정에서 LO를 쿼리할 때 작성자 세부 정보가 표시되는 방식에는 변경 사항이 없습니다.

**참고:** 통합이 공유 강의의 LO API 응답에서 authorNames 필드를 사용하는 경우 업데이트된 작성자 데이터가 수신 계정의 관리자 이름이 이 필드에 나타난다고 가정하는 다운스트림 논리를 위반하지 않는지 확인하십시오.
